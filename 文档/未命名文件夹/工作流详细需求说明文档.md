# 工作流系统（Workflow）落地执行手册 (SSOT 唯一事实来源)

> **核心原则**：本文档为工作流特性的唯一事实来源 (Single Source of Truth, SSOT)。所有关于工作流的开发约束、业务逻辑和任务拆解以此文档为准。**坚决禁止**创建类似 `_v2`、`_final` 这样带有版本号的新副本。所有开发必须严格在 `工作流` 分支上进行。

## 一、 项目集成说明 (基于 README.md 规范)

### 1. 目录与文件约定
- **工作流广场/入口**: 修改 `AI 创作/通用生成.html`
- **编辑器页面**: 新建 `AI 创作/工作流编辑器.html` (强制要求：纯原生 HTML/JS/CSS 实现拖拽式画板，保证零依赖与 0 白屏概率)
- **原料库页面**: 新建 `AI 创作/原料库.html` (明确划分为“视频”、“图片”、“文案”三大模块)

```javascript
    url: 'AI 创作/工作流编辑器.html?id=xxx' 
}, '*');
```

## 二、 核心技术与 UI/UX 实现规范 (开发强制约束)

1. **全局样式底线**: 页面 `body` 背景色必须严格限定为 `#f5f7fa`，padding 为 `8px`。坚决避免使用可能导致滚动条锁死或双滚动条的 `height: 100vh`。
2. **文案工具输入配置**: 文案生成能力必须支持配置『参考图片』(Reference Image)上传、『裂变数量』(Fission Count，1-100条) 设定，以及『文案风格』设置。
3. **文案结果呈现红线**: 文案的生成产物在任何页面（包括结果页、任务预览）都**必须**以“垂直列表（一行一条）”的形式展现，坚决禁止套用图片视频的网格卡片（Grid）布局；另外，在“我的任务”中，文本生成必须是标准任务流列表样式，**严禁**使用聊天窗口（Chat）样式。
4. **动态配置与连线传参**: 工作流配置采用“动态表单”收集输入参数，参数传递必须支持基于 `{{NodeID.Variable}}` 的上下文映射。
5. **节点抽象方式**: 编辑器中的节点，应尽快收口成 `processor` 类型体系。对用户暴露的是可理解的分类（输入、解析、生成、审查、输出），而不是平铺一堆零散的原子步骤。
6. **最终产物追溯 (Traceability)**: 所有合成与生成的最终资产必须附带元数据，强制链接回其对应的 `workflow_id` 与 `task_id`，支持在原料库中无缝反向溯源。
7. **END 节点语义**: 结束节点默认表示“进入任务系统”，工作流跑完后结果天然回到“我的任务”体系中继续被查看、审查、入库、追溯。不需要再把 END 节点设计成复杂的出库/回调路由配置器。

## 三、 详细任务卡片 (Teabition 拆解执行)

### [卡片 01] 通用生成页：双向分发入口与横向精致化卡片
- **目标**: 建立与顶层生产 Tab 同步的工作流入口，并通过“精致化”卡片设计提升专业 SaaS 工具感。
- **执行要求**:
    1. **双向联动 Tab**: 
       - 工作流区域必须具备独立的“文案/图像/视频”切换开关。
       - 必须实现与页面顶层类型切换的**全量同步**（双向互刷），确保用户无论从哪操作，生产上下文均保持一致。
    2. **精致化卡片视觉规范 (落地标准)**:
       - **紧凑型横向布局**: 摒弃传统大图，采用左图标、右文案、底信息的扁平化结构。
       - **图标优化**: 采用 `48x48px` 圆角矩形图标框，底色限定为极浅主题色（如 `rgba(22,119,255,0.08)`），内置根据类型生成的矢量 SVG 图标，禁止文字占位。
       - **文本降噪**: 删除了“用于”等引导词。标题使用 `15px/600` 字号，描述文本使用 `13px` 且行高为 `1.5`，通过对比度拉开层级。
       - **去除物理线**: 严禁在卡片内部使用横向分割线，通过空间留白实现分区。
    3. **布局分层与数量管理**:
       - **置顶 (Pinned)**: 展示 5 个，视觉优先级最高。
       - **我的 (Mine)**: 展示 15 个，包含编辑与副本功能。
       - **广场 (Square)**: 展示 20 个，标题右侧必须配置「查看更多」跳转至独立广场页。
    4. **卡片生命周期菜单**:
       - 鼠标 Hover 时在卡片右下角显式出现 `...` 操作触发器。
       - 菜单项必须包含：`置顶/取消置顶`、`编辑`、`创建副本`、`删除`。

### [卡片 02] 文案裂变：多模态输入与垂直列表展现
- **目标**: 解决“识图写文”和批量文案的结构化展示。
- **执行要求**:
    1. **输入面板**: 增加 `Upload` 组件（参考图）和 `Slider`（设定裂变数量 1-100）及风格选项。
    2. **结果呈现**: 解析 JSON 输出，强制使用**垂直列表（一行一条）**展示文案内容，提供“一键复制”和“入原料库”等内联动作，**彻底废弃原有的 Grid 卡片阵列布局**。

### [卡片 03] 工作流编辑器：真·拖拽式画板生产线
- **目标**: 用户自定义“图-文-视”自动化链路。
- **执行要求**:
    1. **交互底线**: 必须实现“拖拽式、画板式”的节点连线交互。放弃最初设想的“垂直步骤条(Steps)”，提供直观的节点图可视化体验。
    2. **技术栈约束**: 由于 CDN 依赖容易导致白屏崩溃，**必须采用纯原生 HTML/JS/CSS 实现**，保证零依赖与极速加载。
    3. **节点配置**: 点击节点右侧滑出配置表单 Drawer，支持参数实时同步到节点卡片，支持连线拖拽与双击删除连线。

### [卡片 04] 任务中心重构：聚合大区块看板与流式审查 (Streaming Review)
- **目标**: 根据要求“让 我的任务.html 支持工作流聚合卡片，显示任务进度，相关信息，以及人工审核信息和审核页面，以及审核操作按钮”，解决自动工作流产生大量任务导致的列表刷屏问题，实现以“工作流执行批次”为维度的聚合展示与审核闭环。
- **执行要求**:
    1. **专属视图切换**: 在顶部的筛选栏（Filter Bar）新增专门的「工作流」Tab，将常规的单次生成任务与复杂的工作流任务进行物理隔离，避免信息干扰。
    2. **工作流聚合卡片 (Workflow Aggregation Card)**: 
       - 坚决摒弃满屏刷新的单条任务列表。将原本零散的任务以“工作流执行批次”为维度打包展示成单个聚合卡片。
       - **进度可视化**：引入三段式进度条（绿色代表已完成、黄色代表待审核、灰色代表进行中），直观反馈工作流整体健康度和卡点。
       - **聚合数据统计**：直观展示当前工作流的总任务数以及“已完成”、“待审核”、“进行中”的具体数值。
       - **节点追踪**：以标签 (Tag) 形式（如 `AI文案: 10/10`, `底图裂变: 15/20`）追踪并展示内部各个子节点的详细进度。
    3. **沉浸式审查台 (Review Station)**: 
       - **原型对齐**：审查页采用全屏视图，顶部展示工作流节点 Tab（带完成/进行中状态），支持点击切换节点内容。
       - **流式审查网格**：物料以 180px 最小宽度的网格平铺，支持“通过/拒绝”快捷动作，状态以右下角图标（钩/叉）实时反馈。
       - **预览集成 (Feature 1)**：深度集成 `image-preview` 组件，点击卡片进入沉浸式全屏预览模式，支持视频播放与左右导航。
       - **内容溯源 (Feature 3)**：物料卡片 Hover 时必须浮现生成该物料的 Prompt 背景（源信息），确保审核有据可依。
       - **快捷键流 (Feature 5)**：强制支持方向键导航及 `Enter` 通过、`Delete` 拒绝的交互逻辑。
       - **流式流转机制**：单个物料只要被点“通过”或“拒绝”，支持快捷操作。审核完毕（或点击“全部通过”）后，系统立即扣减待审核数量，并将合格产物自动流转至下游节点（触发下一步生成）。

### [卡片 05] 原料库：中间资产的多模态管理与溯源
- **目标**: 统一存储生产线拆解/衍生的输入原料及流转产物（半成品），并支持反查关联。
- **执行要求**:
    1. **分类维度**: 明确提供“图片素材”、“视频素材”、“文案素材”三大标签页 (Tabs)。
    2. **展示形式**: 图片和视频采用卡片网格流（Grid）直观展示视觉，文案采用列表流（List）并提供一键复制文本。
    3. **核心溯源**: 必须在每个产物卡片上展示明确的产物溯源链路（如徽章/标签）。明确标示该原料是“外部导入(手动上传)”还是“产物溯源(由 WF-ID / TSK-ID 产出)”，用户点击溯源标签可追溯工作流任务的上下文。
