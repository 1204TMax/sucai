# 工作流系统（Workflow）落地执行手册 (SSOT 唯一事实来源)

> **核心原则**：本文档为工作流特性的唯一事实来源 (Single Source of Truth, SSOT)。所有关于工作流的开发约束、业务逻辑和任务拆解以此文档为准。**坚决禁止**创建类似 `_v2`、`_final` 这样带有版本号的新副本。所有开发必须严格在 `工作流` 分支上进行。

## 一、 项目集成说明 (基于 README.md 规范)

### 1. 目录与文件约定
- **工作流广场/入口**: 修改 `AI 创作/通用生成.html`
- **编辑器页面**: 新建 `AI 创作/工作流编辑器.html` (强制要求：纯原生 HTML/JS/CSS 实现拖拽式画板，保证零依赖与 0 白屏概率)
- **原料库页面**: 新建 `AI 创作/原料库.html` (明确划分为“视频”、“图片”、“文案”三大模块)
- **任务/结果页**: 修改 `AI 创作/我的任务.html` (实现聚合看板与待审状态) 及相关全部结果页

### 2. 菜单配置 (修改 menu-config.js)
必须精确地放在左侧菜单的『AI 创作』分组下，且必须紧挨在『项目管理』的上方。

## 二、 核心技术与 UI/UX 实现规范 (开发强制约束)

1. **全局样式底线**: 页面 `body` 背景色必须严格限定为 `#f5f7fa`，padding 为 `8px`。
2. **新建工作流弹窗**: 必须提供弹窗采集封面、名称、**主产物类型（文案/图片/视频）**、描述和可见性配置。
3. **文案呈现红线**: 任何产出页面中文案必须以“**垂直列表（一行一条）**”形式展现。在“我的任务”中，文本生成任务必须是标准任务流样式，**严禁使用聊天窗口样式**。
4. **编辑器交互**: 必须实现“**拖拽式、画板式**”的节点连线交互。由于 CDN 稳定性问题，**必须采用纯原生 HTML/JS/CSS 实现**。
5. **最终产物追溯 (Traceability)**: 所有资产必须附带元数据，强制链接回其对应的 `workflow_id` 与 `task_id`，支持在原料库中无缝反向溯源。
6. **审查逻辑**: 必须支持 **Streaming Review (流式审查)**。单个物料通过审核后应立即流转至下游，无需等待批次完成。

## 三、 详细任务卡片 (Teabition 拆解执行)

### [卡片 01] 通用生成页：分发广场与新建工作流弹窗
- **目标**: 建立生产入口，规范工作流创建。
- **执行要求**:
    1. **Tab 切换**: 顶部增加“文案”选项，并支持双向同步类型切换。
    2. **新建弹窗**: 实现封面上传、名称输入、主产物类型选择、描述、私有/公开权限设置。
    3. **布局**: 实现“置顶、我的、广场”三段式层级展示。

### [卡片 02] 文案裂变：多模态输入与垂直列表展现
- **目标**: 解决“识图写文”和批量文案的结构化展示。
- **执行要求**:
    1. **输入配置**: 支持参考图上传、1-100 条裂变数量设定及风格选择。
    2. **结果呈现**: **强制使用垂直列表（一行一条）**展示，废弃 Grid 网格布局。

### [卡片 03] 工作流编辑器：真·拖拽式画板生产线
- **目标**: 用户自定义自动化链路。
- **执行要求**:
    1. **交互**: 节点自由拖拽、桩位连线交互、双击连线删除。
    2. **配置**: 侧边 Drawer 实时配置节点参数，支持变量引用。
    3. **v1.0 节点启用边界**: 长期分类虽然会覆盖 `输入 / 解析 / 生成 / 审查 / 输出`，但当前 v1.0 画布只启用 `输入 / 生成 / 审查 / 输出`，不直接暴露 OCR、拆解等解析节点。
    4. **首批节点范围**: 当前编辑器只保留 6 类节点：`开始`、`文本生成`、`图片生成`、`视频生成`、`人工审查`、`结束`。
    5. **开始节点**: 名称固定为 `开始`；字段包含 `输入（动态新增）`、`目标产出类型`、`最终产出数量（1-100）`、`比例/分辨率`。
    6. **文本生成节点**: 字段包含 `名称`、`模型`、`提示词`、`参考文件`、`生成数量`；其中 `提示词`、`参考文件` 均支持三种来源：`自己填写/上传`、`继承开始节点输入`、`继承其他节点输出`。
    7. **图片生成节点**: 字段包含 `名称`、`模型`、`提示词`、`参考文件`、`生成数量`、`分辨率`、`是否去水印`；其中 `提示词`、`参考文件` 同样支持三种来源。
    8. **视频生成节点**: 字段包含 `名称`、`模型`、`提示词`、`参考文件`、`生成数量`、`分辨率`、`是否去水印`；其中 `提示词`、`参考文件` 同样支持三种来源。
    9. **人工审查节点**: 字段收敛为 `名称`、`审查对象`、`审查标准`、`审查方式`、`是否允许全部通过`；并持续遵守流式审查语义。
    10. **结束节点**: 名称固定为 `结束`；只表达“进入任务系统”语义，不再承载模型、提示词、参考文件、数量、分辨率等生成配置。
    11. **变量引用标准**: 开始节点输入参数应能以 `{{StartNodeID.field_key}}` 形式被后续节点引用；其他生成节点输出应能以 `{{NodeID.output}}` 形式被后续节点引用。

### [卡片 04] 任务中心重构：聚合看板与流式审查台
- **目标**: 解决任务刷屏问题，实现以“执行批次”为维度的聚合。
- **执行要求**:
    1. **聚合卡片**: 展示三段式进度条（已完/待审/进行中）及节点 Tag。
    2. **宏观看板**: 顶部大区块展示进行中、待审核等实时数据统计。
    3. **流式审查**: 进入全屏审查台，通过即流转。

### [卡片 05] 原料库：溯源管理体系
- **目标**: 统一存储原料并支持反向追溯。
- **执行要求**:
    1. **分类**: 视频/图片/文案三大 Tab。
    2. **溯源**: 产物强制外露溯源徽章，记录 WF-ID。
