## 1. 产品定位

工作流是 AI 创作模块中的流程化创作能力，允许用户将文本生成、图片生成、视频生成与人工审核等节点串联为可复用流程。用户可以创建、编辑、试运行、发布、收藏、复制并使用工作流，也可以在任务体系中查看运行进度、处理人工审核、查看全部结果。

工作流系统需要同时满足三类目标：

| 目标 | 说明 |
|---|---|
| 可创建 | 用户能够通过拖拽式编辑器完成工作流搭建，并保存为可复用资产。 |
| 可运行 | 已发布工作流可从通用生成等入口发起任务，并在我的任务中追踪执行状态。 |
| 可治理 | 工作流支持可见性、部门公开、收藏、复制、人工审核与失败原因查看。 |

## 2. 信息架构

工作流归属于「AI 创作」模块，由固定菜单页与动作进入页共同组成。

```text
AI 创作
├── 通用生成
│   ├── 新建工作流弹窗
│   └── 工作流编辑器
├── 我的任务
│   ├── 工作流运行态卡片
│   ├── 人工审核页
│   └── 全部结果页
├── 工作流广场
├── 工作流编辑器
└── 项目管理
```

核心页面职责如下：

| 页面 | 职责 |
|---|---|
| 通用生成 | 承接单点创作，并展示「我的收藏」「我的创建」工作流快捷入口。 |
| 工作流广场 | 展示部门范围内公开工作流，支持筛选、排序、收藏与创建副本。 |
| 工作流编辑器 | 通过拖拽画布配置节点、连线、参数、权限，并完成试运行与发布。 |
| 我的任务 | 融合展示图像、视频、文案与工作流任务，承接任务状态追踪和审核入口。 |
| 全部结果页 | 按任务类型展示已生成结果，并支持下载。 |
| 人工审核页 | 处理工作流中人工审核节点产生的待审核内容。 |

## 3. 全局规则

### 3.1 工作流生命周期

工作流生命周期分为创建、编辑、试运行、发布、使用五个阶段。

| 阶段 | 规则 |
|---|---|
| 创建 | 用户从通用生成页点击「新建工作流」，先填写基础信息，再进入编辑器。 |
| 编辑 | 画布、节点、连线、名称、描述、权限等修改均自动保存。 |
| 试运行 | 用户对工作流做出任意修改后，必须重新试运行。 |
| 发布 | 只有试运行通过后才允许发布。 |
| 使用 | 只有发布后的工作流，才可以在工作流广场和通用生成页展示并被正式使用。 |

### 3.2 权限与可见性

工作流可见性分为「仅自己可见」和「团队公开」。

| 可见性 | 可见范围 |
|---|---|
| 仅自己可见 | 仅创建者本人可见和使用。 |
| 团队公开 | 创建者所属层级部门成员可见；该层级负责人也可见。 |

工作流广场仅展示当前用户所在部门范围内的公开工作流。用户可以看到自己创建的公开工作流，也可以看到同部门及权限范围内他人公开的工作流。

### 3.3 收藏、删除与副本

| 操作 | 规则 |
|---|---|
| 收藏 | 用户可一键收藏，也可一键取消收藏。 |
| 删除 | 删除工作流前必须弹出二次确认弹窗。 |
| 创建副本 | 用户可对广场中的工作流创建副本；副本内容与原工作流完全一致，名称自动追加「 - 副本」。 |

他人创建的工作流在广场中只允许收藏或创建副本，不允许直接编辑或删除。

### 3.4 节点引用规则

开始节点支持定义运行时由用户填写或上传的输入项。中间节点禁止手动上传参考内容，所有参考内容必须通过下拉框引用上游节点输出。

参数面板中，凡是引用上游节点数据的操作，包括插入变量、选择参考文件、选择最终输出内容，均使用统一的级联选择菜单：

| 层级 | 展示规则 |
|---|---|
| 一级 | 展示来源节点。 |
| 二级 | 统一展示「输出结果」，不额外暴露内部变量名。 |
| 不可用状态 | 当上游节点无可用输出时，菜单项置灰不可点击。 |

### 3.5 单值、数组与组合生成

节点输出数量会影响后端数据类型与后续组合生成逻辑。

| 场景 | 规则 |
|---|---|
| 单个输出 | 后端按单值类型处理，如字符串、图片或视频；该类型不在页面展示。 |
| 多个输出 | 本质是多次请求模型，结果按数组类型处理。 |
| 单一文本输入 | 若当前节点输入仅为单一文本，允许用户自由设定生成数量，默认上限为 100。 |
| 数组变量参与生成 | 当提示词插入数组变量，或参考文件引用数组资源时，节点按数组组合逻辑生成。 |
| 生成数量限制 | 用户可选择低于组合总量的生成数量；达到用户设定数量后停止。 |
| 组合任务上限 | 出现数组变量或数组参考资源参与时，系统判定为组合生成任务，最大生成数量动态收紧至 50。 |

每个节点必须在自身结果全部生成完成后，才会将结果一次性传递给下一个节点。

## 4. 通用生成页

文件：`AI 创作/通用生成.html`

### 4.1 页面职责

通用生成页承担两个职责：一是保留原有单点创作能力；二是展示与当前用户相关的工作流快捷入口，包括「我的收藏」和「我的创建」。

### 4.2 工作流展示

工作流展示区域按顶部 Tab 区分文案、图像、视频三类工作流。区域内除标题和「新建工作流」按钮外，按「我的收藏」和「我的创建」分组展示。

| 内容 | 规则 |
|---|---|
| 我的收藏 | 展示用户收藏的工作流；为空时不展示该标题。 |
| 我的创建 | 展示用户创建的工作流；为空时不展示该标题。 |
| 示例工作流 | 每个用户初始在「我的创建」中拥有一个名为「示例」的工作流。 |
| 卡片信息 | 展示标题、描述、封面图、最近编辑时间。 |
| 卡片操作 | 支持编辑、收藏、取消收藏、删除、使用。 |

### 4.3 文案通用生成

文案生成支持模型选择、数量选择、项目选择、提示词工具箱和参考文件能力。用户选择生成数量后，系统按该数量请求模型；例如选择 10 条，即请求 10 次模型。

模型支持多选。模型列表展示模型展示名、模型真实名和能力标签。

当前模型清单：

| 展示名 | 模型真实名 |
|---|---|
| Gemini 3.1 Pro Preview | `gemini-31-pro` |
| Gemini 3 Flash Preview | `gemini-3-flash` |
| DeepSeek V4 Pro | `deepseek-v4-pro` |
| DeepSeek V4 Flash | `deepseek-v4-flash` |
| 豆包 2.0 | `doubao-20` |
| GPT 5.4 | `gpt-54` |
| Claude Sonnet 4.6 | `claude-sonnet-46` |

参考文件入口由所选模型能力决定：

| 场景 | 规则 |
|---|---|
| 单模型选择 | 只展示该模型支持的参考文件类型。 |
| 多模型选择 | 可上传类型取所有已选模型共同支持能力的交集。 |
| 已上传参考文件 | 不允许再选择不支持该文件类型的模型。 |

示例：用户在选中 Gemini 后上传了视频，则不允许再选择不支持视频参考的 DeepSeek。

发起通用生成任务后，系统提示 toast：「任务已开始，您可在我的任务页中查看任务进度。」发起后不自动跳转到我的任务页。

### 4.4 新建工作流

用户点击「新建工作流」时，系统先检测是否存在上一次未完成但已保存的工作流。

| 检测结果 | 处理方式 |
|---|---|
| 存在未完成草稿 | 弹窗提示用户选择「新建工作流」或「进入上次编辑」。 |
| 选择新建工作流 | 删除上次编辑内容，并打开新建工作流弹窗。 |
| 选择进入上次编辑 | 打开工作流编辑器，并恢复上次保存内容。 |
| 不存在草稿 | 直接打开新建工作流弹窗。 |

新建工作流弹窗字段如下：

| 字段 | 规则 |
|---|---|
| 工作流名称 | 必填，200 字以内。 |
| 输出类型 | 必选，支持文案、图片、视频。 |
| 描述 | 必填。 |
| 封面图 | 默认根据工作流名称自动生成，也支持用户上传。 |
| 可见性 | 支持仅自己可见、团队公开。 |

用户完成基础信息填写后，进入工作流编辑器。

## 5. 工作流广场页

文件：`AI 创作/工作流广场.html`

### 5.1 页面职责

工作流广场展示当前用户所在部门范围内的公开工作流，支持按类型聚合、搜索、筛选、排序、收藏和创建副本。

### 5.2 展示与筛选

页面按文案、图像、视频聚合展示工作流，并提供以下检索能力：

| 能力 | 规则 |
|---|---|
| 关键词搜索 | 支持模糊搜索工作流名称、说明、创建者。 |
| 创建者筛选 | 单选下拉；展示同部门成员及其领导姓名；下拉列表支持模糊搜索。 |
| 最近编辑日期 | 支持按最近编辑日期筛选。 |
| 排序 | 支持按使用次数、收藏数量、最近编辑时间升序或降序排序。 |

### 5.3 工作流卡片

广场卡片展示工作流基础信息、创建人、最近编辑时间、收藏数量和使用次数。最近编辑时间前需要展示创建人名字。

卡片操作规则：

| 工作流归属 | 可用操作 |
|---|---|
| 自己创建 | 可查看、收藏、创建副本；如后续支持编辑入口，须遵循权限口径。 |
| 他人创建 | 仅支持收藏和创建副本。 |

## 6. 工作流编辑器页

文件：`AI 创作/工作流编辑器.html`

### 6.1 页面布局

编辑器采用拖拽式画布结构：

| 区域 | 内容 |
|---|---|
| 左上角 | 工作流任务名称。 |
| 左侧 | 节点列表。 |
| 中间 | 无限画布。 |
| 右侧 | 节点配置菜单。 |
| 右上角 | 自动保存时间状态、试运行、发布。 |

新建工作流进入编辑器后，画布默认展示开始节点和结束节点。开始节点与结束节点始终存在，任何状态下都不可删除。

### 6.2 画布交互

用户可以从左侧节点列表拖拽节点到画布，并通过连线建立节点关系。线条默认从左到右流转，每个节点左侧为输入，右侧为输出。

画布支持自动保存。以下操作均触发自动保存：

| 操作类型 | 示例 |
|---|---|
| 画布操作 | 拖拽节点、连线、删除节点。 |
| 节点配置 | 修改模型、提示词、参考文件、比例、数量等配置。 |
| 工作流信息 | 修改任务名称、任务描述、权限。 |

拖入节点时，如画布中已存在同名节点，系统自动追加编号后缀，例如 `文本处理-1`、`文本处理-2`。

### 6.3 试运行与发布

用户进入编辑器后，只要对节点配置、连线、节点本身、任务名称、任务配置信息或权限做出任意修改，就必须重新点击试运行。

发布规则：

| 条件 | 结果 |
|---|---|
| 未试运行 | 不允许发布。 |
| 试运行失败 | 不允许发布。 |
| 试运行通过 | 允许发布。 |

只有发布后的工作流，才可正式使用，并进入工作流广场和通用生成页展示区域。

### 6.4 参数面板通用交互

文本生成、图片生成、视频生成三类核心节点共用底层交互组件，包括带标签检索的模型选择器，以及支持变量胶囊混排的提示词编辑器。

| 组件 | 规则 |
|---|---|
| 模型选择器 | 支持下拉搜索、能力标签展示，可附带轻量配置，如开启思考模式。 |
| 提示词编辑器 | 支持纯文本与变量胶囊混排；变量胶囊展示为蓝色，带来源标识和拖拽手柄。 |
| 参考文件 | 明确区分本地上传与引用上游文件变量。 |
| 上传入口 | 点击上传文字时唤起本地系统文件窗口。 |
| 引用入口 | 点击选择区域时唤起工作流内置级联菜单。 |

在比例或分辨率配置旁，需要增加 Tooltip：「若作为后续节点的参考图，建议选择较高分辨率」。

### 6.5 图像节点配置规则

配置图像比例时，系统根据比例类型展示不同配置项。

| 比例类型 | 配置规则 |
|---|---|
| 固定比例 | 如 16:9，同时展示清晰度选择，支持 1k、2k、4k。 |
| 自定义比例 | 展示宽高双输入框，用于精确定义分辨率。 |

用户输入自定义宽高像素时，系统实时校验合法性。若不合法，对应输入框进入错误状态，并在输入区域下方展示错误提示；当前值不得用于试运行或发布。用户修正为合法值后，错误状态与提示自动消失。

### 6.6 视频节点配置规则

视频生成节点不支持思考模式，也不支持自定义比例。视频比例和时长均通过下拉列表选择。

视频生成新增「参考模式」切换：

| 参考模式 | 规则 |
|---|---|
| 智能参考 | 使用统一参考对象选择。 |
| 首尾帧 | 分离展示「首帧」与「尾帧」参考对象选择。 |

视频和图像节点均不支持自动去水印等非相关能力。

### 6.7 节点清单

左侧节点列表仅展示 4 个可拖拽节点：文本生成、图片生成、视频生成、人工审核。开始节点和结束节点不在左侧节点列表展示，但默认存在于画布。

| 节点 | 输入输出 | 配置项 | 说明 |
|---|---|---|---|
| 开始节点 | 无输入，仅输出 | 多个运行时输入项 | 定义工作流运行时由用户填写或上传的初始参数；不支持修改节点名称。 |
| 结束节点 | 仅输入，无输出 | 输出类型、输出内容 | 声明最终结果；输出类型为文案、图片、视频之一；输出内容从上游节点输出中选择；不支持修改节点名称。 |
| 文本生成 | 有输入，有输出 | 名称、模型、思考模式、提示词、参考文件、生成数量 | 用于生成创意文案、脚本或描述文本。 |
| 图片生成 | 有输入，有输出 | 名称、模型、提示词、参考文件、比例/分辨率、生成数量 | 用于生成图片结果。 |
| 视频生成 | 有输入，有输出 | 名称、模型、提示词、参考模式、参考文件、比例、时长、生成数量 | 用于生成视频结果。 |
| 人工审核 | 有输入，有输出 | 节点名称 | 暂停流程，等待人工审核通过或拒绝产物后再继续。 |

### 6.8 节点设计原则

节点应遵循单一职责原则。复杂业务逻辑应拆分为多个原子节点；未来复杂行业算法引擎可封装为行业专用 Processor。

## 7. 我的任务页

文件：`AI 创作/我的任务.html`

### 7.1 页面职责

我的任务页融合展示图像、视频、文案和工作流任务，并提供任务状态追踪、结果查看、失败详情查看和人工审核入口。

### 7.2 导航与状态提示

我的任务页新增文案 Tab，并在原有“进行中任务”和“任务已生成未查看任务”之外，新增“待审核任务”。

| 规则 | 说明 |
|---|---|
| 待审核导航展示 | 存在待人工审核任务时，展示“待审核任务”导航。 |
| 点击导航 | 自动定位到对应任务区域。 |
| 侧边栏菜单 | 同步展示待审核状态及对应数量。 |

### 7.3 状态文案

任务状态文案统一如下：

| 场景 | 状态口径 |
|---|---|
| 待人工审核 | 原“开始审核”或“审查中”统一为「待人工审核 (N个)」。 |
| 审核按钮 | 中间流程节点操作按钮统一为「点击审核」。 |
| 通用生成结束状态 | 使用「已终止、已完成、已失败」。其中原“已成功”改为“已完成”。 |
| 工作流结束状态 | 使用「已终止、已完成」。 |

工作流任务不再根据内部失败情况区分全部失败或部分失败来定义结束状态。只要工作流已经无法继续运行，无论是中间节点全部失败导致无法继续，还是流程全部进入结束节点，均统一认为该工作流任务「已完成」。

### 7.4 工作流任务卡片

普通状态下，工作流任务卡片展示工作流标签、工作流任务名称、生成结果、开始时间和状态。底部按钮与通用生成任务卡片保持一致。

失败处理规则：

| 场景 | 规则 |
|---|---|
| 存在生成失败 | 展示「查看详情」按钮。 |
| 点击查看详情 | 弹窗按节点聚合展示真实失败原因。 |
| 全部失败 | 卡片仅展示「全部失败」，具体原因在详情弹窗中查看。 |
| 失败原因口径 | 生成节点失败原因沿用图像、视频通用生成失败原因口径。 |

待人工审核状态下，卡片切换为原型中的待人工审核样式，并展示开始节点、结束节点，以及当前触发的待人工审核节点。

当同时存在多个待人工审核节点时，卡片增加第 3 个节点，用于展示剩余审核节点数量。第 2 个节点仍展示当前按顺序需要先审核的节点；用户点击审核后，按顺序逐个审核。

### 7.5 文案任务卡片

文案任务的任务类型、卡片类型、状态和按钮交互与图像、视频任务一致，差异仅在结果展示。

| 规则 | 说明 |
|---|---|
| 预览数量 | 卡片中最多展示前 4 条文案。 |
| 少于 4 条 | 按实际生成数量展示。 |
| 入口 | 文案区域右下角展示文案数量和「查看全部」按钮。 |

## 9. 人工审核页

### 9.1 入口与页面结构

用户在我的任务页点击「点击审核」后，进入人工审核页。

页面结构如下：

| 区域 | 规则 |
|---|---|
| 顶部 | 展示工作流任务名称。 |
| 节点按钮区 | 以按钮形式展示该任务中的人工审核节点。 |
| 默认节点 | 默认进入用户刚才点击「点击审核」所对应的可审核节点。 |
| 状态筛选 | 提供「全部」「可审核」「已通过」「已拒绝」。 |
| 默认筛选 | 默认进入「可审核」。 |

### 9.2 节点可点击规则

人工审核节点只有在待审核内容已生成完成，并且该节点所需 Input 全部进入后，才触发「待审核」。

| 节点状态 | 交互 |
|---|---|
| 待审核内容已就绪 | 节点按钮可点击。 |
| 待审核内容仍在生成中 | 节点按钮置灰，不允许点击。 |
| 鼠标悬停置灰按钮 | 提示「待审核内容还未就绪」。 |

### 9.3 审核内容展示

不同内容类型采用不同展示方式：

| 类型 | 展示规则 |
|---|---|
| 图片 | 支持列表查看和大图预览。 |
| 视频 | 支持列表查看和播放预览。 |
| 文案 | 与全部结果页中的文案展示形式一致，按一条一条展示。 |

文案过长时默认展示两行，用户可点击「查看全部」展开，再点击「收起」收回。

### 9.4 审核操作

每个内容下方均展示通过和拒绝按钮。点击通过表示该内容审核通过，点击拒绝表示该内容审核不通过。点击后即刻生效，且无论通过或拒绝，均不可撤回。

页面右上角提供「全部通过」和「全部拒绝」按钮。点击后，仅对当前已经生成好的内容执行一次性审核操作。

### 9.5 预览操作

图片或视频可点开查看大图或播放。预览态底部仍需提供明确的通过和拒绝按钮，同时提供「上一张」「下一张」切换能力。
