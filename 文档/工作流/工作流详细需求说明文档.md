## 一、项目概述

### 1.1 项目背景

- 当前 AI 创作模块已经具备文案、图像、视频等单点生成能力，但业务实际生产经常需要多步骤串联，例如“爆款素材拆解 → 文案衍生 → 图片生成 → 人工审查 → 视频合成 → 输出结果”。
- 仅依赖通用生成会导致用户反复手动复制提示词、上传素材、筛选结果和发起下一步任务，生产链路长、重复操作多、过程不可追踪。
- 工作流能力的目标不是把所有生成过程变成完全黑盒，而是提供一套可配置、可复用、可审查、可追踪的半自动生产流水线。
- 工作流系统需要同时服务两类用户：
  - 工作流创建者：懂业务、会配置流程，希望沉淀生产方法。
  - 工作流使用者：只想快速使用成熟流程，不需要理解全部节点细节。
- 因此，工作流系统既要支持较复杂的画布式搭建，也要在使用和任务查看阶段尽量降低认知成本。

### 1.2 项目目标

构建一套围绕 AIGC 生产链路的工作流系统，让业务、产品、设计、研发能够基于同一套定义推进页面、交互、状态和接口设计：

1. 支持用户创建、编辑、发布和使用工作流。
2. 支持工作流按主产物类型归类为文案、图像、视频。
3. 支持工作流绑定项目，用于读取项目素材、历史文案、产品信息等上下文。
4. 支持工作流通过节点画布编排多步骤生产链路。
5. 支持人工审查节点作为工作流中的 Check Point。
6. 支持工作流任务进入“我的任务”，并按最终产物类型融合展示。
7. 支持工作流运行中展示简化流程状态，让用户知道当前是否需要自己处理。
8. 支持工作流结果回流到结果页、任务中心和后续素材/原料管理链路。
9. 支持工作流广场作为独立页面，用于发现、筛选、收藏和使用更多工作流。
10. 统一页面范围、字段口径、状态流转、按钮动作、跳转关系和设计约束。

### 1.3 核心设计原则

| 原则 | 说明 |
|---|---|
| 使用入口轻量 | 通用生成页只展示「我的收藏」和「我的创建」，不承载去广场的显性跳转入口。 |
| 广场去中心化 | 工作流广场主要用于发现他人的公开工作流，不重复展示用户「我的创建」内容。 |
| 创建与使用分离 | 创建者在编辑器里理解节点细节；使用者在运行时只填写必要输入并查看结果。 |
| 变量驱动参考 | 中间节点禁止手动上传文件，强制要求引用上游节点输出的变量作为参考内容。 |
| 运行状态聚合 | 我的任务不暴露完整复杂节点图，只展示开始、当前处理点、待人工审核、结束等简化状态。 |
| 审查语义明确 | 明确区分状态描述（如“待人工审核”）与行动指令（如“点击审核”）。 |
| 结果导出便捷 | 文案等文本类大批量产出支持一键导出为 Excel/CSV 表格。 |
| 节点职责单一 | 避免开发“万能节点”，通过原子节点（如专门的拆解节点）组合实现复杂业务。 |
| 页面风格统一 | 所有页面保持统一、稳定、紧凑的系统视觉风格。 |

## 二、系统架构

### 2.1 页面层级结构

工作流系统归属于「AI 创作」模块，由固定菜单页和动作进入页共同组成：

```text
AI 创作（固定菜单）
├── 通用生成                  ← 单点创作入口 + 我的收藏/我的创建工作流
│   ├── 新建工作流弹窗         ← 采集工作流基础信息
│   └── 工作流编辑器           ← 画布式搭建
├── 我的任务                  ← 图像 / 视频 / 文案任务融合展示
│   ├── 工作流运行态卡片       ← 展示当前流程状态和审核入口
│   ├── 审查台                ← 处理人工审核任务
│   └── 全部结果页             ← 查看任务产物及导出
├── 工作流广场                ← 独立广场，发现与收藏他人的工作流
├── 工作流编辑器              ← 画布搭建，支持自动编号命名与通用能力开关
├── 原料库                    ← 资产管理
└── 项目管理                  ← 上下文管理
```

### 2.2 文件与菜单约定

| 页面 | 文件 | 是否固定菜单 | 说明 |
|---|---|---|---|
| 通用生成 | `AI 创作/通用生成.html` | 是 | 快捷入口包含「我的收藏」与「「我的创建」 |
| 我的任务 | `AI 创作/我的任务.html` | 是 | 运行态展示「待人工审核」与「点击审核」动作 |
| 工作流广场 | `AI 创作/工作流广场.html` | 是 | 独立广场，移除「官方创建/我创建的」筛选，仅展示他人工作流 |
| 工作流编辑器 | `AI 创作/工作流编辑器.html` | 可动作进入 | 支持节点自动编号、去水印开关、禁用中间节点上传 |
| 全部结果页 | `AI 创作/全部结果-*.html` | 否 | 文本结果支持「导出 Excel」 |

## 三、页面详细设计

### 3.1 通用生成页

文件：`AI 创作/通用生成.html`  
入口：左侧导航「通用生成」；首页快捷入口  
职责：承接单点创作，展示「我的收藏」与「我的创建」快捷入口。

#### 3.1.1 页面结构

| 区域 | 说明 |
|---|---|
| 我的收藏 | 展示用户从广场收藏的工作流，标题对仗。 |
| 我的创建 | 展示用户自己创建的工作流，不再展示去广场的跳转链接。 |

#### 3.1.2 功能点

- 工作流快捷区跟随顶部全局类型切换。
- **标题优化**：明确为「我的收藏」和「我的创建」。
- **入口精简**：去掉原本放在「我的」右侧的「查看工作流广场」跳转。
- **展示统一**：卡片底部不再展示创建者信息（因均为用户关联内容，默认隐藏以保持简洁）。
- 收藏 / 取消收藏、删除操作实时生效。
- **通用生成页-文案**：
  - 模型改为多选，多选后在模型后面选择数量。
  - 模型列表增加搜索模型名称输入框，并在选项旁展示该模型支持的特征标签（如图片、视频、音频等）。

### 3.2 工作流广场页

文件：`AI 创作/工作流广场.html`  
入口：左侧导航「工作流广场」  
职责：展示公开的他人工作流，支持筛选、搜索、收藏。

#### 3.2.1 页面结构

- **筛选区简化**：移除「官方创建 / 我创建的」来源快捷筛选 Tab。
- **列表纯净化**：仅展示非用户本人的工作流，避免与通用生成页内容重复。

#### 3.2.2 功能点

- **身份标识**：卡片统一显示真实的创建者名字，不做“我”的特殊判断，降低逻辑复杂度。
- **组合筛选**：
  - **基础搜索**：支持按关键词搜索工作流名称、说明、项目信息。
  - **创建者筛选**：提供带搜索功能的下拉列表，支持快速检索并选择系统内的创建者。
  - **日期筛选**：支持选择创建日期的起止区间，进行范围过滤。
- **排序**：支持按使用次数、创建时间、收藏数量进行升降序排列。
- **空状态**：当无匹配的工作流时，展示大面积空白区域，并统一固定提示文案为「此处什么都没有」。

### 3.3 工作流编辑器页

文件：`AI 创作/工作流编辑器.html`  
职责：通过拖拽式画布配置工作流。

#### 3.3.1 画布与节点交互

- **极简卡片视觉**：画布中的节点卡片采用纯白底色、柔和阴影与圆角设计。节点类型通过左侧带底色的方块 Icon 区分，不再使用顶栏横向色条。
- **自动编号命名**：向画布拖入节点时，若同名节点已存在，系统自动增加后缀（如 `文本处理-1`、`文本处理-2`）。
- **引用逻辑强制化**：
  - 开始节点：支持定义手动上传项（图片/视频/文本等）。
  - **中间节点**：**禁止手动上传参考内容**。所有参考内容必须通过下拉框选择“之前节点输出的文件”。
- **通用能力内聚**：
  - 视频和图像节点均不支持思考模式与自动去水印等非相关能力。
  - 在比例/分辨率配置旁增加 Tooltip 提示：「若作为后续节点的参考图，建议选择较高分辨率」。

#### 3.3.3 节点参数面板交互规范

- **模型与提示词组件通用化**：文本生成、图像生成、视频生成三大核心节点共用一套底层交互组件。包括带标签检索的模型选择器、支持“变量胶囊”拖拽混排的富文本编辑器。
- **参考文件分离操作**：明确区分“本地直接上传”与“引用上游文件变量”的触点。点击上传文字唤起本地系统文件窗口；点击选择区域唤起工作流内置级联菜单。
- **数量限制与排列组合规则**：
  - 若当前节点输入仅为单一文本，允许用户自由设定生成数量（默认上限 100）。
  - 若提示词内插入了“输出变量”（代表可能的数组组合），或引用了前置的参考文件资源，系统自动将其判定为全组合生成任务。此时生成数量组件会自动出现说明，并将最大限制动态收紧至 50，避免生成爆炸。

- **图像分辨率与清晰度联动**：配置图像生成比例时，若选择固定比例（如16:9），则附加「清晰度」选择（1k/2k/4k）；若选择自定义，则提供宽高双输入框供用户精确定义分辨率。

- **视频生成专属配置与约束**：视频生成模型不支持思考模式；新增「参考模式」切换，支持「智能参考」与「首尾帧」两种模式，后者将分离出独立的“首帧”与“尾帧”参考对象选择；视频比例不支持自定义，仅可通过下拉选择（如 16:9）；时长同样限定为下拉列表选择（5秒、10秒等）。

- **基础节点限制**：开始节点与结束节点作为流程基础锚点，不支持修改节点名称。
- **结束节点产物配置**：结束节点要求明确声明最终的“输出类型”（图片、视频、文本之一），其“输出内容”同样复用标准的级联选择菜单交互，不再使用传统的原生下拉框。

#### 3.3.4 节点设计原则

- **模型选择器**：支持下拉搜索与能力标签（如“图片”、“视频”）展示，并可附带轻量级的开关配置（如“开启思考模式”）。
- **变量级联选择**：参数面板内所有引用上游节点数据的操作（如插入变量、选择参考文件），均采用统一的级联菜单交互：首级选择来源节点，次级选项统一且极简地只显示“输出结果”四个字，不附加具体的变量名称。当上游节点无可用输出时，菜单项表现为文字置灰（不可点），而不破坏列表结构。
- **流式提示词编辑**：提示词输入框支持纯文本与“变量胶囊”混排。插入的变量表现为蓝色胶囊（带有来源标识），胶囊左侧提供拖拽手柄，用户可将其自由拖拽到输入框内任意文本位置，实现灵活拼接。

#### 3.3.2 节点设计原则

- **单一职责**：复杂的业务逻辑拆分为多个原子节点。
- **行业封装**：未来将复杂的李强算法引擎等封装为行业专用 Processor。

### 3.4 我的任务页

文件：`AI 创作/我的任务.html`

#### 3.4.1 任务卡片文案优化

- **状态文案**：原“开始审核”或“审查中”统一调整为「**待人工审核 (N个)**」。
- **动作按钮**：中间流程节点的操作按钮文案改为「**点击审核**」。

#### 3.4.2 任务结束状态口径

- 当任务为**通用生成**时，结束状态分为：**已终止、已完成、已失败**。
- 其中，需要特别说明：上一版本中的“已成功”，在当前版本中统一调整为“**已完成**”。
- 当任务为**工作流**时，结束状态分为：**已终止、已完成**。
- 对于工作流任务，不再根据任务内部是否存在失败、是否为全部失败或部分失败来定义结束状态。
- 只要工作流已经无法继续运行，无论是因为中间某个节点全部失败而无法继续，还是因为流程已经全部进入结束节点，都统一认为该工作流任务已经“**已完成**”。

### 3.5 全部结果页

文件：`AI 创作/全部结果-统一版.html`

#### 3.5.1 导出能力

- **导出 Excel**：针对文本（文案/脚本）类结果，在顶部操作栏增加「导出 Excel」按钮。
- **格式要求**：导出文件包含序号、内容等字段，方便运营直接粘贴到其他业务表格。

## 四、开发优先级与验收标准

### 4.1 v1.0 补充验收项

1. 通用生成页标题是否为「我的收藏」和「我的创建」，是否去掉了去广场入口。
2. 广场页是否仅显示他人工作流，卡片是否统一显示创建者名字。
3. 编辑器内新增同名节点是否自动编号。
4. 编辑器中间节点是否已禁用手动上传，比例设置是否有分辨率建议提示。
5. 我的任务卡片文案是否准确（待人工审核 / 点击审核）。
6. 文案结果页是否具备「导出 Excel」功能。
