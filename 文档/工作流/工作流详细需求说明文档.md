# 工作流系统总需求文档

> 本文档为 AI 创作模块中「工作流」能力的详细需求说明文档，面向产品、设计、研发、测试和后续 AI Coding 执行使用。
>
> **SSOT 原则**：本文档是工作流功能的唯一事实来源。后续不再创建 `_v2`、`_final`、`最新版` 等副本文档。工作流相关需求变化必须直接更新本文档。
>
> **当前版本口径**：本文档已吸收 2026年4月27日工作流确认会议结论，并补充 2026年4月28日「我的任务」文案任务卡片展示规则。若早期文档中存在“工作流广场放在通用生成页内”“物料级流式审查”“任务卡片展示 4 个 loading 位”“多条文案合并成一段编号文本”等旧口径，以本文档为准。

## 一、项目概述

### 1.1 项目背景

- 当前 AI 创作模块已经具备文案、图像、视频等单点生成能力，但业务实际生产经常需要多步骤串联，例如“爆款素材拆解 → 文案衍生 → 图片生成 → 人工审查 → 视频合成 → 输出结果”。
- 仅依赖通用生成会导致用户反复手动复制提示词、上传素材、筛选结果和发起下一步任务，生产链路长、重复操作多、过程不可追踪。
- 工作流能力的目标不是把所有生成过程变成完全黑盒，而是提供一套可配置、可复用、可审查、可追踪的半自动生产流水线。
- 工作流系统需要同时服务两类用户：
  - 工作流创建者：懂业务、会配置流程，希望沉淀生产方法。
  - 工作流使用者：只想快速使用成熟流程，不需要理解全部节点细节。
- 因此，工作流系统既要支持较复杂的画布式搭建，也要在使用和任务查看阶段尽量降低认知成本。

### 1.2 项目目标

构建一套围绕 AIGC 生产链路的工作流系统，让业务、产品、设计、研发能够基于同一套定义推进页面、交互、状态和接口设计：

1. 支持用户创建、编辑、发布和使用工作流。
2. 支持工作流按主产物类型归类为文案、图像、视频。
3. 支持工作流绑定项目，用于读取项目素材、历史文案、产品信息等上下文。
4. 支持工作流通过节点画布编排多步骤生产链路。
5. 支持人工审查节点作为工作流中的 Check Point。
6. 支持工作流任务进入“我的任务”，并按最终产物类型融合展示。
7. 支持工作流运行中展示简化流程状态，让用户知道当前是否需要自己处理。
8. 支持工作流结果回流到结果页、任务中心和后续素材/原料管理链路。
9. 支持工作流广场作为独立页面，用于发现、收藏和使用更多工作流。
10. 统一页面范围、字段口径、状态流转、按钮动作、跳转关系和设计约束。

### 1.3 核心设计原则

| 原则 | 说明 |
|---|---|
| 使用入口轻量 | 通用生成页只展示收藏和我的工作流，不承载完整广场，避免干扰单点创作。 |
| 广场独立承载 | 工作流广场作为独立菜单页，负责发现、筛选、收藏和使用公开工作流。 |
| 创建与使用分离 | 创建者在编辑器里理解节点细节；使用者在运行时只填写必要输入并查看结果。 |
| 单步多步兼容 | 工作流通常用于多步骤生产链路，但也允许封装单步最佳实践、提示词模板和固定业务场景。 |
| 运行状态聚合 | 我的任务不暴露完整复杂节点图，只展示开始、当前处理点、待审查、结束等简化状态。 |
| 审查节点可多个 | 工作流中可配置多个审查节点，系统不限制只能有一个人工审查。 |
| 审查按节点聚合 | 审查不按每个物料碎片化打扰用户，而是按审查节点或批次聚合，让用户一次性处理。 |
| 结果按类型归类 | 工作流任务不单独开 Tab，按最终产物类型进入文案、图像、视频任务流。 |
| 文案列表展示 | 文案结果必须使用垂直列表，一行一条，不使用网格卡片，不使用聊天窗口样式。 |
| 页面风格统一 | 所有页面遵循 `共享样式/usage.md`，页面背景 `#f5f7fa`，body padding 为 `8px`。 |
| 原生优先稳定 | 页面原型必须使用纯原生 HTML/CSS/JS 实现，避免 CDN 依赖导致白屏。 |
| 直接打开可看 | 当前原型文件应支持直接双击或通过浏览器打开本地 HTML 查看，不要求启动本地服务作为预览前提。 |

## 二、系统架构

### 2.1 页面层级结构

工作流系统归属于「AI 创作」模块，由固定菜单页和动作进入页共同组成：

```text
AI 创作（固定菜单）
├── 通用生成                  ← 单点创作入口 + 工作流快捷入口
│   ├── 新建工作流弹窗         ← 采集工作流基础信息
│   └── 工作流编辑器           ← 创建后进入画布编辑
├── 我的任务                  ← 图像 / 视频 / 文案任务融合展示
│   ├── 工作流运行态卡片       ← 展示当前流程状态和审查入口
│   ├── 审查台                ← 处理工作流人工审查任务
│   └── 全部结果页             ← 查看任务完整产物
├── 工作流广场                ← 独立广场，用于发现、筛选、收藏、使用工作流
├── 工作流编辑器              ← 工作流搭建与编辑页，通常由新建/编辑动作进入
├── 原料库                    ← 后续增强页，承接多模态原料与溯源，不作为本轮强依赖
└── 项目管理                  ← 项目数据来源和上下文管理
```

### 2.2 文件与菜单约定

| 页面 | 文件 | 是否固定菜单 | 说明 |
|---|---|---|---|
| 通用生成 | `AI 创作/通用生成.html` | 是 | 单点创作入口，展示收藏和我的工作流快捷入口 |
| 我的任务 | `AI 创作/我的任务.html` | 是 | 统一任务中心，工作流任务按最终产物类型融合展示 |
| 工作流广场 | `AI 创作/工作流广场.html` | 是 | 独立广场，承接工作流发现与收藏 |
| 工作流编辑器 | `AI 创作/工作流编辑器.html` | 可固定，也可动作进入 | 工作流画布搭建页 |
| 审查台 | `AI 创作/我的任务.html?view=review...` 或独立承接页 | 否 | 从任务卡片进入，处理审查任务 |
| 全部结果页 | `AI 创作/全部结果-*.html` | 否 | 查看完整结果 |
| 原料库 | `AI 创作/原料库.html` | 可固定 | 后续承接原料与溯源，本轮不作为工作流 MVP 必须依赖 |

### 2.3 核心对象模型

```text
Workflow（工作流定义）
├── 基础信息
├── 项目归属
├── 主产物类型
├── 封面信息
├── 权限与可见性
├── 节点定义
├── 连线定义
├── 运行输入表单定义
├── 发布状态
└── 创建/更新时间

WorkflowNode（工作流节点）
├── 节点 ID
├── 节点类型
├── 节点名称
├── 输入桩/输出桩
├── 配置参数
├── 上游变量引用
├── 测试状态
└── 运行状态

WorkflowRun（工作流运行任务）
├── 任务 ID
├── 工作流 ID
├── 所属项目
├── 主产物类型
├── 用户运行输入
├── 当前运行状态
├── 节点运行快照
├── 审查任务聚合
├── 结果统计
└── 创建/完成时间

ReviewTask（审查任务）
├── 审查任务 ID
├── 所属 WorkflowRun
├── 所属审查节点
├── 待审内容集合
├── 已通过内容集合
├── 已拒绝内容集合
├── 未运行到数量
└── 审查状态

WorkflowResult（工作流产物）
├── 产物 ID
├── 产物类型
├── 文件/文本内容
├── 来源工作流 ID
├── 来源任务 ID
├── 来源节点 ID
├── 审查状态
├── 入库状态
└── 生成时间
```

### 2.4 关键数据流

```text
通用生成页 ──新建工作流──→ 新建工作流弹窗 ──创建──→ 工作流编辑器
工作流编辑器 ──保存草稿/发布──→ 工作流定义

工作流广场 ──收藏──→ 通用生成页「收藏」
工作流广场 ──使用──→ 工作流运行输入表单 ──提交──→ 我的任务

通用生成页「我的/收藏」──使用──→ 工作流运行输入表单 ──提交──→ 我的任务

我的任务 ──工作流运行中──→ 工作流运行态卡片
我的任务 ──需要审查──→ 审查台 ──通过/拒绝──→ 工作流继续运行
我的任务 ──运行完成──→ 最终结果卡片 ──查看全部──→ 全部结果页

工作流结果 ──附带 WF-ID / TSK-ID / Node-ID──→ 后续原料库、素材库、物料库或结果页追溯
```

核心约束：

- 通用生成页负责快捷使用，不负责完整广场浏览。
- 工作流广场负责发现和收藏，不负责复杂编辑。
- 工作流编辑器负责搭建，不负责任务流结果展示。
- 我的任务负责运行状态和结果，不负责修改工作流结构。
- 审查台负责人工处理，不负责展示完整编辑画布。

## 三、页面详细设计

### 3.1 通用生成页

文件：`AI 创作/通用生成.html`  
入口：左侧导航「通用生成」；首页快捷入口  
职责：承接文案、图像、视频的单点创作，同时展示与当前创作类型相关的工作流快捷入口。

#### 3.1.1 页面结构

通用生成页由两部分组成：

| 区域 | 说明 |
|---|---|
| 单点创作区 | 原有文案、图像、视频生成能力，由顶部全局类型切换控制 |
| 工作流快捷区 | 展示收藏工作流和我创建的工作流，跟随顶部全局类型切换 |

工作流快捷区不再包含内部「文案 / 图像 / 视频」二级切换，避免和顶部全局切换产生割裂。

「新建工作流」入口应优先放在「我的」区域标题右侧。若新用户「我的」列表为空，保留「我的」标题和「去工作流广场」入口，并在下方展示无工作流空状态；空状态中的「新建工作流」按钮可比标题栏按钮更醒目。

当当前创作类型下收藏工作流数量为 0 时，不展示「收藏」标题和收藏区域，避免空区域占用页面。

#### 3.1.2 数据字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 当前创作类型 | enum | 文案 / 图像 / 视频，由页面顶部全局切换控制 |
| 收藏工作流列表 | Workflow[] | 当前用户收藏的工作流，按当前创作类型过滤 |
| 我的工作流列表 | Workflow[] | 当前用户创建的工作流，按当前创作类型过滤 |
| 工作流名称 | string | 卡片主标题 |
| 工作流描述 | string | 卡片说明 |
| 主产物类型 | enum | 文案 / 图像 / 视频 |
| 所属项目 | string | 工作流绑定项目 |
| 创建时间 | datetime | 工作流创建时间 |
| 封面 | object | 自动文字封面或用户上传封面 |
| 收藏状态 | boolean | 是否已收藏 |
| 可见性 | enum | 私有 / 团队公开 / 广场公开 |

#### 3.1.3 功能点

- 支持在同一页面完成单点文案、图像、视频生成。
- 工作流快捷区跟随顶部全局类型切换，只展示当前类型可用的工作流。
- 工作流快捷区只展示「收藏」和「我的」，不展示完整「广场」。
- 「收藏」用于快速访问从工作流广场收藏的工作流。
- 「我的」用于快速访问当前用户创建的工作流。
- 当用户想查看更多工作流时，通过「去工作流广场」跳转到独立工作流广场页。
- 当用户点击「新建工作流」时，打开新建工作流弹窗。
- 当用户点击某个工作流卡片的「使用」时，进入该工作流的运行输入表单或直接开始运行。
- 当用户点击某个工作流卡片的「编辑」时，打开工作流编辑器。
- 工作流卡片不展示「图片 / 文案 / 视频」类型标签，类型由顶部全局切换表达；卡片底部对应位置展示创建者和创建时间。
- 工作流卡片封面 / 图标区域应比早期小图标方案更突出，便于用户快速识别工作流。
- 收藏 / 取消收藏应真实改变当前页面列表：收藏后进入「收藏」区域，取消收藏后从「收藏」区域移除；原型演示可采用运行时状态，刷新后恢复默认数据。
- 删除应真实从当前页面列表移除对应工作流；原型演示不要求持久化删除，刷新后可恢复默认数据。
- 单点创作区用于解决用户即时的单点生产诉求，可以支持多模态输入，但不应做成通用大模型聊天页的重复产品。
- 工作流快捷区用于让用户使用已经封装好的业务生产方法。工作流可以是多步链路，也可以是单步模板化工具，关键区别在于它是可复用、可管理、可沉淀的生产资产。

#### 3.1.4 新建工作流弹窗

新建工作流弹窗用于采集创建工作流所需的最小基础信息。创建成功后进入工作流编辑器。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 名称 | string | 是 | 工作流名称，用于卡片、任务和编辑器标题 |
| 所属项目 | enum | 是 | 选择项目，如消效乐、跑酷大冒险等 |
| 主产物类型 | enum | 是 | 文案 / 图像 / 视频，只能选择一个 |
| 描述 | textarea | 否 | 说明该工作流适用场景和能力 |
| 可见性 | enum | 是 | 私有 / 团队公开 / 广场公开 |
| 封面 | object | 否 | 默认自动生成，也支持上传图片 |

封面规则：

- 封面不是第一必填项，不应让用户误以为创建前必须上传图片。
- 默认封面采用“名称关键字 + 色块”的方式生成，参考钉钉群封面逻辑。
- 系统从工作流名称中提取 4-6 个字作为封面文字。
- 默认色块可自动分配，也允许用户修改颜色。
- 用户可选择上传自定义封面，但上传封面是可选项。

#### 3.1.5 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 文案 / 图像 / 视频 | 切换当前创作类型 | 单点创作区和工作流快捷区同步切换 |
| 新建工作流 | 创建新的工作流 | 打开新建工作流弹窗 |
| 去工作流广场 | 查看更多公开工作流 | 打开 `工作流广场.html` |
| 使用 | 使用当前工作流 | 打开运行输入表单或创建工作流任务 |
| 编辑 | 编辑当前工作流 | 打开 `工作流编辑器.html` |
| 收藏 / 取消收藏 | 更新收藏状态 | 当前工作流加入或移出收藏列表 |
| 创建并进入画布 | 提交新建弹窗 | 创建工作流草稿并打开编辑器 |
| 取消 | 关闭新建弹窗 | 不创建工作流 |

### 3.2 工作流广场页

文件：`AI 创作/工作流广场.html`  
入口：左侧导航「工作流广场」；通用生成页「去工作流广场」  
职责：展示公开或可见范围内的工作流，支持筛选、搜索、收藏、使用和查看。

#### 3.2.1 页面结构

| 区域 | 说明 |
|---|---|
| 搜索筛选区 | 按关键词、产物类型、来源、热度等筛选工作流 |
| 工作流列表区 | 以卡片展示工作流信息 |
| 空状态 | 无匹配工作流时展示空状态和引导 |

工作流广场是“大卖场”定位，不和通用生成页混在一起。

#### 3.2.2 数据字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 关键词 | string | 搜索工作流名称、描述、创建者 |
| 主产物类型 | enum | 全部 / 文案 / 图像 / 视频 |
| 来源 | enum | 全部 / 官方 / 团队 / 我创建的 / 他人公开 |
| 排序 | enum | 最近更新 / 使用最多 / 收藏最多 |
| 工作流名称 | string | 卡片主标题 |
| 工作流描述 | string | 卡片说明 |
| 创建者 | string | 工作流创建人 |
| 所属项目 | string | 工作流绑定项目或适用项目 |
| 使用次数 | number | 被使用次数 |
| 收藏次数 | number | 被收藏次数 |
| 更新时间 | datetime | 最近更新 |
| 收藏状态 | boolean | 当前用户是否收藏 |
| 可见性 | enum | 团队公开 / 广场公开 |

#### 3.2.3 功能点

- 支持浏览官方、团队、自己和他人公开的工作流。
- 支持按主产物类型筛选，便于用户快速找到可用于文案、图像或视频生产的工作流。
- 支持收藏工作流，收藏后在通用生成页「收藏」区展示。
- 支持直接使用工作流，使用后创建工作流运行任务并进入我的任务。
- 支持查看工作流详情，了解输入要求、输出类型、适用场景、创建者和使用数据。
- 支持空状态展示，避免列表为空时布局塌陷。

#### 3.2.4 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 搜索 | 按关键词检索工作流 | 刷新工作流列表 |
| 类型筛选 | 按主产物类型筛选 | 刷新工作流列表 |
| 来源筛选 | 按工作流来源筛选 | 刷新工作流列表 |
| 排序切换 | 调整排序规则 | 按新规则展示列表 |
| 收藏 | 收藏当前工作流 | 加入通用生成页收藏区 |
| 取消收藏 | 取消收藏当前工作流 | 从收藏区移除 |
| 使用 | 使用当前工作流 | 打开运行输入表单或创建任务 |

#### 3.3.3 节点类型规划

| 节点类型 | 说明 | v1.0 要求 |
|---|---|---|
| 开始节点 | 定义运行时用户需要填写的表单输入 | 必须支持 |
| 结束节点 | 定义最终输出类型和结果归属 | 必须支持 |
| 文本处理节点 | 文案生成、改写、总结、提取卖点等 | 必须支持 |
| 图像处理节点 | 图片生成、图片改写、图生图等 | 必须支持 |
| 视频处理节点 | 视频生成、视频合成、视频改写等 | 必须支持 |
| 素材拆解节点 | 输入爆款视频，拆出音轨、片段、字幕和结构 | v1.0 重点支持 |
| 素材衍生节点 | 基于拆解结构或输入素材做裂变 | v1.0 重点支持 |
| 自动混剪节点 | 将通过审查的素材按规则组合为成片 | v1.0 重点支持 |
| 人工审查节点 | 暂停自动流转，由人工通过/拒绝 | 必须支持 |

通用能力内聚原则：

- 去水印、等比例裁剪、视频超分、格式转换等通用能力不作为独立节点优先暴露。
- 这类能力应作为图像处理节点或视频处理节点中的配置项存在。
- 避免让用户为了高频小动作连接大量低价值节点。

工作流步数原则：

- 工作流不强制必须多步。若一个节点已经足以承载稳定业务场景，也可以形成单步工作流。
- 单步工作流适合沉淀提示词模板、固定输入表单、项目上下文和默认参数。
- 多步工作流适合沉淀需要拆解、衍生、审查、合成等环节的复杂生产链路。
- 无论单步还是多步，都必须通过开始节点定义运行输入，并通过结束节点定义最终产物类型。

#### 3.3.4 画布交互

- 支持从左侧节点库拖拽节点到画布。
- 支持节点在画布上自由拖动。
- 支持画布左右、上下平移。
- 支持画布缩放，缩放用于查看全貌和快速定位。
- 支持从输出桩拖拽到输入桩建立连线。
- 支持拖动已有连线端点重新连接。
- 支持双击连线删除。
- 支持删除节点，并同步删除关联连线。
- 支持点击节点打开右侧配置 Drawer。
- 支持开始节点和结束节点默认存在。
- 开始节点和结束节点原则上各只能保留一个。

#### 3.3.5 开始节点规则

开始节点定义“用户使用这个工作流时需要填什么”。

| 字段 | 类型 | 说明 |
|---|---|---|
| 输入项名称 | string | 如“裂变方向”“参考图片”“目标数量” |
| 输入项类型 | enum | 文本 / 图片 / 视频 / 文件 / 数字 / 选择器 |
| 是否必填 | boolean | 使用工作流时是否必须填写 |
| 默认值 | any | 可选默认值 |
| 说明 | string | 给使用者看的输入提示 |

开始节点必须支持目标产出数量配置。目标数量用于后续循环、质量校验和结果统计。

#### 3.3.6 结束节点规则

- 结束节点定义最终输出类型。
- 最终输出类型必须与新建工作流时选择的主产物类型一致。
- 一个工作流只能有一个主产物类型：文案、图像或视频。
- 工作流内部可以产生中间文案、图片、视频，但最终输出只能归到一个主产物类型下。
- 结束节点走完后，工作流运行任务状态变为已完成，并在我的任务中展示结果。

#### 3.3.7 人工审查节点规则

- 工作流中允许配置多个人工审查节点。
- 人工审查节点可以出现在文案生成后、图片生成后、视频合成前等不同阶段。
- 如果多个上游节点连到同一个人工审查节点，代表这些上游产物需要一起被审查。
- 同一个人工审查节点必须等所需上游内容都准备好后，才进入可审查状态。
- 审查通过的内容进入下游节点。
- 审查拒绝的内容不进入下游节点，后续可按配置决定是否补生成、跳过或终止。
- 审查节点不应把每个物料拆成独立任务频繁打扰用户，应在任务中心聚合展示。

#### 3.3.8 测试与发布

| 状态 | 说明 |
|---|---|
| 未测试 | 节点或工作流尚未执行测试 |
| 测试中 | 正在运行测试 |
| 测试通过 | 测试输入下执行成功 |
| 测试失败 | 测试输入下执行失败，需要调整配置 |

发布规则：

- 工作流必须保存后才能发布。
- 关键节点测试通过后才允许发布。
- 开始节点、结束节点、主链路连通性必须校验通过。
- 存在孤立节点时允许保存草稿，但发布前必须提示。
- 发布后可被自己使用；公开后可进入团队或广场范围。

#### 3.3.9 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 保存草稿 | 保存当前画布配置 | 工作流状态为草稿 |
| 测试节点 | 测试当前节点 | 更新节点测试状态 |
| 试运行 | 按开始节点表单试运行整个工作流 | 生成测试结果 |
| 发布 | 发布工作流 | 状态变为已发布 |
| 下架 | 下架已发布工作流 | 状态变为已下架 |
| 添加节点 | 从节点库拖拽新增节点 | 画布新增节点 |
| 删除节点 | 删除当前节点 | 删除节点和关联连线 |
| 删除连线 | 删除当前连线 | 连线移除 |
| 关闭配置 | 关闭右侧 Drawer | 返回画布视图 |

### 3.4 我的任务页

文件：`AI 创作/我的任务.html`  
入口：左侧导航「我的任务」；工作流运行后自动进入或由通知跳转  
职责：集中查看 AI 创作任务，工作流任务按最终产物类型融合到文案、图像、视频任务流中。

#### 3.4.1 页面结构

| 区域 | 说明 |
|---|---|
| 类型 Tab | 文案 / 图像 / 视频，工作流任务按最终产物类型进入对应 Tab |
| 筛选区 | 按任务状态、来源、项目、时间等筛选 |
| 任务列表 | 展示普通生成任务和工作流任务 |
| 左侧悬浮通知 | 聚合生成中、审查中、未查看等任务入口 |
| 底部输入区 | 保留普通任务再生成或新生成入口 |

#### 3.4.2 工作流任务展示规则

工作流任务分两类展示：

| 工作流任务状态 | 展示方式 |
|---|---|
| 未完成 / 运行中 / 审查中 | 展示工作流运行态卡片 |
| 已完成 | 展示普通结果卡片 |

运行态卡片结构：

```text
工作流标签 + 工作流名称                         主产物类型 + 输出规格

开始  →  当前节点（开始审查/生成中）  →  待审查（N）  →  结束

查看详情 / 重新编辑 / 停止生成                         状态 + 时间
```

设计要求：

- 卡片视觉必须和其他任务卡片统一，不允许使用脱离系统的灰底线框图风格。
- 老板手绘图仅作为信息结构参考，不作为视觉样式稿。
- 当前需要用户处理的审查节点高亮展示，并提供「开始审查」按钮。
- 如果当前没有可审查内容，则展示「生成中」或「生产中」，不展示可点击的审查按钮。
- 未运行到的审查任务可以在界面上简写为「待审查（N）」，但产品和接口语义上表示“后续还未运行到的审查节点/审查批次”，不是已经可操作的待办。
- 当前可操作审查和未运行到审查必须在状态上区分清楚，避免用户误以为所有审查都已经可以处理。
- 已完成工作流不再展示流程图，直接展示最终结果。

#### 3.4.3 文案任务卡片展示规则

文案任务需要同时表达两层信息：

1. 外层任务卡片代表一次生成请求，例如“帮我重写 20 条小红书文案”。
2. 内层文案结果行代表本次任务产出的每一条可被单独处理的文案。

因此，文案任务不能只在任务卡片中展示一个合并后的段落，也不能采用聊天消息气泡样式。应使用“任务卡片 + 文案结果行列表”的结构。

文案任务卡片结构：

```text
任务来源 + 生成要求摘要                         模型 + 项目/场景

01  文案内容摘要...                         已完成   复制 / 编辑 / 再生 / 采用
02  文案内容摘要...                         已完成   复制 / 编辑 / 再生 / 采用
03  文案内容摘要...                         已完成   复制 / 编辑 / 再生 / 采用

共计产出 20 条文案 · 每条文案可单独处理          查看全部

重新编辑 / 再次生成 / 查看全部                 状态 + 时间
```

文案结果行字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| 行序号 | number | 当前预览列表中的展示序号，从 1 开始 |
| 文案内容 | string | 单条文案正文，列表态单行截断，详情态可展开全文 |
| 单条状态 | enum | 生成中 / 已完成 / 已失败 / 已采用 / 已入库 |
| 操作按钮 | array | 复制、编辑、再生、采用等单条操作 |
| 来源任务 ID | string | 所属任务，用于回溯生成请求 |
| 文案结果 ID | string | 单条文案唯一标识，用于单条再生、采用、入库 |

状态展示规则：

| 任务状态 | 文案结果行展示 |
|---|---|
| 等待中 | 可展示 1-4 条等待占位行，提示任务排队中 |
| 生成中 | 已产出的文案展示为真实结果行；未产出的数量用骨架行或生成中占位展示 |
| 已完成 | 展示前 3-4 条真实文案结果行，底部展示总产出数量和查看全部入口 |
| 部分失败 | 成功文案正常展示；失败文案展示失败状态，可提供单条再生 |
| 已失败 | 若无任何产出，展示失败占位；若已有部分产出，则按部分失败处理 |
| 已中止 | 保留已产出的文案结果，未完成部分不再继续展示生成中 |

操作规则：

| 操作 | 作用范围 | 说明 |
|---|---|---|
| 复制 | 单条文案 | 将当前文案复制到剪贴板，不影响任务状态 |
| 编辑 | 单条文案 | 打开单条文案编辑入口，编辑结果应保留与原任务的关系 |
| 再生 | 单条文案 | 只重生成当前这一条，不重新生成整批文案 |
| 采用 | 单条文案 | 标记该文案被当前业务采用，可进入后续投放、素材或脚本链路 |
| 查看全部 | 整个任务 | 进入全部结果页，展示本任务所有文案 |
| 再次生成 | 整个任务 | 使用原任务输入重新发起一批文案生成 |
| 重新编辑 | 整个任务 | 回到生成输入表单，允许修改原始需求后重新提交 |

设计要求：

- 文案结果行应是紧凑横条，不做大卡片，避免一条文案占用过多垂直空间。
- 单条操作按钮必须跟随文案结果行展示，不能只放在任务卡片底部，否则用户无法判断操作对象。
- 预览列表默认展示 3-4 条；数量较多时通过「查看全部」进入完整结果。
- 文案内容在任务卡片内默认单行截断，移动端可放宽为 2 行截断。
- 状态标签必须使用有文字的标签，例如“已完成”“生成中”“失败”，不使用纯色圆点。
- 文案任务仍然属于“我的任务”统一任务卡片体系，外层卡片的底部状态、时间、重新编辑、再次生成等操作保持与其他任务一致。

#### 3.4.4 数据字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 任务 ID | string | 工作流运行任务唯一标识 |
| 工作流 ID | string | 来源工作流定义 |
| 工作流名称 | string | 卡片标题 |
| 主产物类型 | enum | 文案 / 图像 / 视频 |
| 输出规格 | string | 如 1920×1080、9:16、文案条数等 |
| 所属项目 | string | 任务所属项目 |
| 当前状态 | enum | 等待中 / 生成中 / 审查中 / 已完成 / 已失败 / 已中止 |
| 当前节点 | string | 当前运行到的节点 |
| 可审查数量 | number | 当前需要用户处理的审查数量 |
| 未运行到审查数 | number | 后续还未运行到的审查节点数量 |
| 已完成审查数 | number | 已处理审查数量 |
| 创建时间 | datetime | 任务创建时间 |
| 完成时间 | datetime | 任务完成时间 |
| 输出结果 | WorkflowResult[] | 最终产物列表 |

#### 3.4.5 工作流任务状态流转

```text
等待中 → 生成中 → 审查中 → 生成中 → 已完成
等待中 → 生成中 → 已完成
等待中 / 生成中 / 审查中 → 已中止
等待中 / 生成中 / 审查中 → 已失败
已失败 → 重新编辑 / 重试 → 等待中
```

状态说明：

| 状态 | 说明 | 用户感知 |
|---|---|---|
| 等待中 | 任务已提交，尚未开始执行 | 可以查看详情、停止生成 |
| 生成中 | 自动节点正在运行 | 展示生产中，不要求用户处理 |
| 审查中 | 至少一个审查节点已准备好，需要用户处理 | 展示开始审查入口 |
| 已完成 | 结束节点已完成，最终结果已生成 | 展示结果，不展示流程图 |
| 已失败 | 工作流执行失败 | 展示失败状态和重试/查看详情 |
| 已中止 | 用户主动停止或系统中止 | 展示已中止状态 |

#### 3.4.6 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 开始审查 | 当前有可审查任务时展示 | 打开审查台 |
| 查看详情 | 查看工作流任务详情 | 打开任务详情或运行详情 |
| 重新编辑 | 基于当前工作流进入编辑器 | 打开工作流编辑器 |
| 停止生成 | 停止当前运行任务 | 二次确认后状态变为已中止 |
| 查看全部 | 查看完整结果 | 打开全部结果页 |
| 重试 | 对失败任务重新发起 | 任务重新进入等待中 |
| 入库 | 将结果进入对应资产库 | 创建入库动作或任务 |
| 下载 | 下载结果 | 创建下载动作或直接下载 |

### 3.5 审查台

文件：可由 `AI 创作/我的任务.html?view=review&taskId=...` 承接，后续也可拆独立页  
入口：我的任务 → 工作流运行态卡片 → 开始审查  
职责：集中处理工作流人工审查节点产生的待审内容。

#### 3.5.1 页面结构

| 区域 | 说明 |
|---|---|
| 任务信息区 | 展示工作流名称、审查节点名称、任务状态 |
| 审查统计区 | 待审、已通过、已拒绝、未运行到等统计 |
| 内容审查区 | 展示当前待审内容 |
| 操作区 | 通过、拒绝、批量通过、提交等 |

#### 3.5.2 数据字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 审查任务 ID | string | 审查任务唯一标识 |
| 工作流任务 ID | string | 所属 WorkflowRun |
| 审查节点 ID | string | 所属人工审查节点 |
| 审查节点名称 | string | 如“审查A”“文案初审”“图片终审” |
| 待审内容 | WorkflowResult[] | 当前需要审查的内容 |
| 已通过内容 | WorkflowResult[] | 已通过内容 |
| 已拒绝内容 | WorkflowResult[] | 已拒绝内容 |
| 未运行到数量 | number | 后续还未到达审查节点的数量 |
| 审查状态 | enum | 未开始 / 可审查 / 审查中 / 已完成 |

#### 3.5.3 审查状态流转

```text
未运行到 → 可审查 → 审查中 → 已完成
可审查 / 审查中 → 已跳过（仅特定配置允许）
```

内容级状态：

```text
待审 → 通过
待审 → 拒绝
拒绝 → 撤回到待审
通过 → 撤回到待审（仅下游未消费时允许）
```

#### 3.5.4 审查逻辑

- 审查台可以展示一个或多个同时可处理的审查任务，但默认聚焦当前触发入口对应的审查任务。
- 如果同一工作流中存在多个并行审查点同时 ready，可以在审查台中切换处理。
- 如果审查节点之间存在前后依赖，下游审查不能修改上游已被消费的审查结果。
- 用户完成当前审查后，系统根据工作流配置继续执行下游节点。
- 系统通知或左侧悬浮提醒应按“审查节点 ready”进行聚合提醒，不按单个产物逐条提醒。
- 若一个工作流需要生产 50 个产物，不应给用户制造 50 个独立审查通知；用户更需要一次进入审查台后集中处理。
- 审查不是聊天，不使用聊天窗口样式。
- 文案审查内容使用垂直列表展示。
- 图片和视频审查可使用卡片或网格，但操作必须清晰可见。

#### 3.5.5 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 通过 | 单项通过 | 当前内容进入通过集合 |
| 拒绝 | 单项拒绝 | 当前内容进入拒绝集合 |
| 批量通过 | 批量通过选中项 | 选中项进入通过集合 |
| 批量拒绝 | 批量拒绝选中项 | 选中项进入拒绝集合 |
| 撤回 | 撤回已处理结果 | 返回待审，受下游消费状态限制 |
| 提交审查 | 完成当前审查节点处理 | 推动工作流进入下游 |
| 返回任务 | 返回我的任务 | 回到任务列表 |

## 四、工作流状态总览

### 4.1 工作流定义状态

```text
草稿 → 已发布 → 已下架
已下架 → 已发布
草稿 → 已删除
已下架 → 已删除
```

| 状态 | 说明 | 可操作 |
|---|---|---|
| 草稿 | 未发布，只创建者可编辑和测试 | 编辑、测试、发布、删除 |
| 已发布 | 可被使用，按权限可见 | 使用、编辑、下架、收藏 |
| 已下架 | 不再允许新使用，历史任务不受影响 | 编辑、重新发布、删除 |
| 已删除 | 逻辑删除 | 不展示 |

### 4.2 工作流运行任务状态

```text
等待中 → 生成中 → 审查中 → 生成中 → 已完成
生成中 → 已完成
等待中 / 生成中 / 审查中 → 已失败
等待中 / 生成中 / 审查中 → 已中止
已失败 → 重试 → 等待中
```

### 4.3 节点运行状态

| 状态 | 说明 |
|---|---|
| 未开始 | 节点尚未被触发 |
| 等待上游 | 上游节点未完成 |
| 运行中 | 当前节点正在执行 |
| 待审查 | 人工审查节点已准备好 |
| 已通过 | 审查节点完成且有通过结果 |
| 已完成 | 节点执行完成 |
| 已失败 | 节点执行失败 |
| 已跳过 | 因条件或配置跳过 |

### 4.4 审查任务状态

| 状态 | 说明 |
|---|---|
| 未运行到 | 工作流还没有执行到该审查节点 |
| 可审查 | 审查节点所需内容已准备好 |
| 审查中 | 用户正在处理或已有部分处理 |
| 已完成 | 审查节点处理完成 |
| 已跳过 | 特定配置允许跳过 |

### 4.5 结果状态

| 状态 | 说明 |
|---|---|
| 生成中 | 结果仍在生成 |
| 待审 | 结果进入审查节点，等待人工处理 |
| 已通过 | 审查通过，可进入下游 |
| 已拒绝 | 审查拒绝，不进入下游 |
| 已完成 | 最终产物完成 |
| 已失败 | 单个结果生成失败 |
| 已入库 | 已进入素材/物料/原料库 |

## 六、质量控制与同质化

### 6.1 同质化检测原则

- 工作流生成的最终产物在入库前必须支持同质化检测。
- 若生成物料与库内已有素材重复率大于 80%，系统需标记为“同质化过高”。
- 同质化过高的产物不能自动入库，必须进入人工确认或待操作流程。
- 同质化检测不阻塞工作流编辑器本身，只影响运行结果入库链路。

### 6.2 结果处理

| 检测结果 | 处理方式 |
|---|---|
| 合格 | 允许进入后续入库或使用流程 |
| 同质化过高 | 标记风险，要求人工确认 |
| 检测失败 | 展示检测失败，可重试或人工处理 |

## 七、文案、图片、视频展示规则

### 7.1 文案结果

- 文案结果必须使用垂直列表。
- 每条文案单独成行，支持复制、编辑、单条再生、采用、通过/拒绝等操作。
- 禁止使用图片式网格卡片展示文案。
- 在我的任务中，文本生成任务必须是标准任务流列表样式，禁止使用聊天窗口样式。
- 如果一次任务生成多条文案，任务卡片内必须展示多个独立文案横条，而不是把多条文案合并成一段编号文本。
- 单条文案横条必须明确操作对象，按钮与文案同行展示；任务级操作如重新编辑、再次生成、查看全部仍放在卡片底部。

### 7.2 图片结果

- 图片结果可使用缩略图网格。
- 缩略图应支持预览、下载、入库、审查操作。
- 生成中可以使用统一 loading 占位。

### 7.3 视频结果

- 视频结果展示缩略图、时长、播放入口。
- 视频预览应支持播放，不只展示静态图。
- 视频处理中的音轨应作为独立对象在数据结构中保留。

当前页面默认采用单文件 HTML 进行原型与页面承载，运行方式依赖系统框架以 iframe 方式加载。

- HTML：页面结构与业务内容承载。
- CSS：页面样式与布局控制。
- JavaScript：页面内状态、按钮动作、页面间 `postMessage` 通信。
- 菜单配置：通过 `menu-config.js` 维护固定菜单与页面映射。
- 框架加载：通过系统主框架 `index.html` 统一承载页面、菜单、标签页与跳转。
- 工作流编辑器：必须使用纯原生 HTML/CSS/JS，不引入不稳定 CDN 依赖。

## 十、验收标准

### 10.1 通用生成页验收

- 顶部文案/图像/视频切换能同步影响工作流快捷区。
- 工作流快捷区只展示收藏和我的，不展示完整广场。
- 有明确入口跳转到工作流广场。
- 新建工作流弹窗字段完整，封面不是第一强制项。

### 10.2 工作流广场验收

- 菜单中存在工作流广场入口。
- 页面可搜索、筛选、收藏、使用工作流。
- 收藏后的工作流能在通用生成页收藏区体现。

### 10.3 工作流编辑器验收

- 可以拖拽节点。
- 可以连线、删除连线、删除节点。
- 可以配置节点参数。
- 可以定义开始节点输入。
- 可以定义结束节点主产物类型。
- 可以保存、测试、发布。

### 10.4 我的任务验收

- 工作流任务按最终产物类型进入文案、图像或视频 Tab。
- 运行中工作流展示流程状态卡。
- 流程状态卡视觉风格与普通任务卡一致。
- 可审查时展示开始审查入口。
- 完成后展示最终结果，不再展示流程图。
- 文案任务卡片内的多条文案必须拆成独立横条展示。
- 每条文案横条必须有清晰的单条操作入口，如复制、编辑、再生、采用。
- 文案任务生成中状态必须同时表达“已产出数量”和“剩余仍在生成”，不能只展示空白或单一句“生成中”。

### 10.5 审查台验收

- 可展示待审、已通过、已拒绝统计。
- 可单项通过/拒绝。
- 可批量通过/拒绝。
- 提交后工作流能继续下游。
