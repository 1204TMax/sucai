# 量化投放系统 · 项目文档

> 本文档是项目级入口，只记录全局背景、工作方式、行为约束和文件导航。具体需求以对应需求文档为准。

## 一、项目背景

本项目是一个面向 AI 创作、素材管理和投放生产链路的静态原型系统，当前通过 GitHub Pages 部署。

- 线上地址：`https://1204tmax.github.io/sucai/`
- 部署来源：GitHub Pages -> `main` 分支 -> `/` 根目录
- 页面承载：主框架 `index.html` 通过 iframe 加载各业务页面
- 菜单配置：`menu-config.js`
- 代码开发指南：`README.md`

## 二、项目目标

- 保持 AI 创作、素材管理、系统设置等模块结构清晰。
- 新增页面时遵循现有 iframe 加载方式和菜单映射方式。
- 页面风格遵循共享样式和设计规范。
- 工作流等复杂能力以需求文档作为唯一事实来源，避免多版本文档并行。

- **前端 UI 防遮挡红线**：在开发下拉菜单（Dropdown）、级联选择器（Cascader）、提示框（Tooltip）或弹窗（Modal）等悬浮类 UI 组件时，**严禁使用局部 `position: absolute` 并挂载在有 `overflow: hidden/auto` 属性的父容器内**。必须采用全局挂载方案（如将 DOM 节点挂载到 `<body>` 根层级），并结合 `position: fixed` 或计算绝对坐标，从根本上杜绝弹出层被外部容器截断、遮挡的经典问题。

## 四、工作流程

### A1：需求与原型确认

1. 用户提出需求或指定正在进行的需求。
2. AI 读取 `文档/项目.md` 和当前需求文档。
3. AI 梳理已确认事项和待确认事项。
4. 用户确认后，再进入页面原型或代码改造。

### A2：代码实现与交付

1. AI 读取 `README.md`、相关页面、`共享样式/usage.md` 和 `共享样式/design-system.md`。
2. AI 给出改造清单和风险点。
3. 用户确认后再修改代码。
4. 完成后更新必要文档和过程记录。
