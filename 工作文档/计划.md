# 工作流任务卡片改造计划

## 背景与目标
本次聚焦 `AI 创作/我的任务.html` 中的工作流任务卡片，目标不是继续做一套独立的工作流视觉，而是把工作流卡片重新收回到统一任务卡片体系：

- 非待审查状态下，工作流卡片直接像普通文案 / 图像 / 视频任务卡片一样展示产物加载态或结果态。
- 只有任意人工审查节点触发时，卡片中间区才切换成抽象节点流，突出当前审查动作。
- 工作流卡片展示状态收敛为 4 类：`生成中`、`待审查`、`已完成`、`已中止`。
- 中途失败或部分失败不再作为 workflow 卡片主状态；只要没有节点继续工作，就归到“已完成”，差异通过“预计数量 / 已产出数量”表达。
- workflow 在文案 / 图像 / 视频三类上都要补齐统一规格口径：文案看数量，图像看数量 + 比例 + 分辨率，视频看数量 + 比例 + 时长。

## 实施要点

### 1. workflow 展示态归一
- 新增 workflow 卡片 view-model 层，不直接依赖裸 `task.status`。
- 归一规则：
  - 有激活人工审查节点 → `待审查`
  - 原始中止 → `已中止`
  - 仍有节点工作 / 排队 → `生成中`
  - 其余全部 → `已完成`

### 2. 非待审查态复用普通任务卡
- workflow 非待审查状态不再展示抽象节点图。
- 直接复用普通卡片的文案 / 图像 / 视频输出区、loading 占位和查看全部入口。

### 3. 待审查态保留抽象节点流
- 固定展示 3 / 4 个节点：
  - 开始
  - 当前审查节点（带节点名和醒目“开始审查”按钮）
  - `待审查(N)`（可选）
  - 结束
- 节点间保留从左到右动效箭头。

### 4. 统一规格与数量口径
- 文案：数量
- 图像：数量 + 比例 + 分辨率
- 视频：数量 + 比例 + 时长
- workflow 统一展示“已产出 / 预计数量”，不再通过失败态表达产出不足。
- workflow 的“已产出 / 预计数量”不再放在卡片头部 meta 中，统一放到右下角状态与日期之间，避免头部信息过重。
- 任务卡片头部不再展示项目参数（如 `护肤品推广`）。

### 5. 底部按钮样式与动作口径对齐
- workflow 卡片底部按钮继续复用普通任务卡按钮体系。
- 对齐圆角、边框、hover、布局与查看全部按钮样式。
- workflow 卡片头部不再展示「使用提示词」按钮。
- workflow 四类状态的操作口径如下：
  - 生成中：查看详情 / 重新编辑 / 查看全部 / 停止生成
  - 待审查：开始审查 / 查看详情 / 重新编辑 / 查看全部 / 停止生成
  - 已中止：查看详情 / 再次生成 / 重新编辑 / 查看全部
  - 已完成：查看详情 / 再次生成 / 重新编辑 / 查看全部

  - 视频显示视频缩略图 / loading / 查看全部

- workflow 原始 `failed` / `partialFailed` 不再出现失败态专卡
- 结果数量与查看全部跳转正确
