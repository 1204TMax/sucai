# 我的任务需求文档

## 1. 页面基本信息
- **页面名称**：我的任务
- **所属模块**：素材管理
- **文件路径**：`素材管理/我的任务.html`
- **页面定位**：素材管理模块内统一的任务中心，用于集中查看上传、下载、视频合成三类异步任务。
- **边界说明**：该页面与 `AI 创作/我的任务.html` 不是同一个页面；本页面仅承载素材管理模块相关任务。

## 2. 页面结构
页面整体采用单卡片结构，不保留额外顶部标题说明卡片，内容直接从任务面板开始，以减少重复信息占位。

### 2.1 顶部任务类型切换
- 页面顶部使用一级 Tab 切换四类任务：`待操作`、`上传`、`下载`、`视频合成`。
- `待操作` 位于首位，用于聚合所有需要人工干预的任务。
- Tab 右侧显示对应任务数量。
- 切换 Tab 后：
  - 筛选项联动切换；
  - 表格列头联动切换；
  - 统计摘要联动切换；
  - 保留各自独立的筛选状态。

### 2.2 筛选区
- 筛选区位于 Tab 下方。
- 筛选控件必须保持**单行紧凑排列**，默认不换行；当视口宽度不足时，允许横向滚动，不应将筛选项自动挤到第二行。
- 所有基础控件高度统一严格执行 **32px**。

#### 待操作视图筛选项
- 时间范围：开始时间、结束时间
- 任务/文件名称搜索
- 任务来源：全部 / 上传 / 视频合成
- 重置筛选（右侧对齐）

#### 上传视图筛选项
- 时间范围：开始时间、结束时间
- 文件名称搜索
- 文件类型：全部 / 视频 / 图片
- 文件属性：全部 / 物料 / 素材
- 所属系列/产品
- 重置筛选（右侧对齐）
...
### 2.4 待操作视图 (新增)
#### 任务模型
- **定义**：从“上传”和“视频合成”任务中自动提取出的异常或待办状态。
- **聚合逻辑**：
    - **上传来源**：包含状态为 `同质化过高`、`上传失败` 的条目（含任务包及子文件）。
    - **视频合成来源**：包含状态为 `合成失败`、`待审核` 的条目。
- **列表字段**：
    - 任务/文件名称
    - 来源类型（上传 / 视频合成）
    - 所属系列 / 产品
    - 进度
    - 状态
    - 提交时间
    - 操作

## 3. 上传任务视图
- 时间范围：开始时间、结束时间
- 文件名称搜索
- 文件类型：全部 / 视频 / 图片
- 文件属性：全部 / 物料 / 素材
- 所属系列/产品
- 重置筛选（右侧对齐）

#### 视频合成视图筛选项
- 时间范围：开始时间、结束时间
- 任务名称 / 策略名称搜索
- 合成状态：全部 / 排队中 / 合成中 / 部分失败 / 已完成 / 已终止
- 输出比例：全部 / 9:16 / 1:1 / 16:9
- 所属系列/产品
- 重置筛选（右侧对齐）

### 2.3 任务统计与摘要
- **已删除**：原有的统计标签（Summary Chips）已移除，以减少视觉噪音并释放纵向空间。

## 3. 上传任务视图

### 3.1 任务模型
- 上传任务需同时支持：
  - **单文件上传任务**；
  - **上传包任务**：一次批量上传的多个文件归属同一上传包。
- 上传包需支持在列表中**展开 / 收起**，展开后查看包内单文件任务明细。

### 3.2 列表字段
上传视图列表需展示以下字段：
- 文件名称
- 文件类型（视频 / 图片）
- 文件属性（物料 / 素材）
- 文件大小
- 上传目标文件夹
- 所属系列 / 产品
- 上传进度
- 状态
- 提交时间
- 操作

### 3.3 状态与操作
#### 单文件上传任务状态
- `同质化检测中`：操作为 `暂停`、`删除`
- `同质化过高`：操作为 `查看同质化内容`、`继续上传`、`删除`
- `上传中`：操作为 `暂停`、`删除`
- `上传失败`：操作为 `重新开始`、`删除`
- `上传成功`：操作为 `查看详情`

#### 同质化内容分析 (Drawer)
- 点击「查看同质化内容」后，从右侧滑出侧边弹窗。
- **展示内容**：
    - 上部展示「当前检测素材」的预览图。
    - 下部展示「同质化率 > 80%」的其他素材列表。
    - 所有素材点击后均支持**全屏大图预览**。
- **交互规范**：点击遮罩或关闭按钮收起；保留极简视觉，移除冗余的文本解释字段。

#### 上传包状态
- `上传中`：操作为 `暂停`、`删除`、`展开/收起`
- `已完成`：操作为 `展开/收起`
- **注意**：原有的「新增策略」快捷入口已从本页面移除。

## 4. 下载任务视图

### 4.1 任务模型
- 下载任务以单文件任务为主。
- 当前阶段无需在下载视图中实现下载包的展开层级。

### 4.2 列表字段
下载视图列表需展示以下字段：
- 文件名称
- 文件类型（视频 / 图片）
- 文件属性（物料 / 素材）
- 文件大小
- 所属文件夹
- 所属系列 / 产品
- 下载进度
- 状态
- 提交时间
- 操作

### 4.3 状态与操作
- `下载中`：操作为 `暂停`、`删除`
- `下载失败`：操作为 `重新下载`、`删除`
- `下载成功`：操作为 `查看`、`打开所在文件夹`

## 5. 视频合成视图

### 5.1 任务模型
- 视频合成指根据合成策略，由多个物料混剪形成新的素材视频的过程。
- 合成任务需采用**合成包**模型承载：例如“100 个视频合成”作为一个包任务。
- 合成包支持**展开 / 收起**，展开后查看包内单条结果视频。
- 页面展示以任务中心视角为主，不在本页展开完整状态机细节；底层流程仍以飞书中的合成策略状态机为准。

### 5.2 列表字段
视频合成视图列表需展示以下字段：
- 合成包 / 策略
- 输入物料 / 目标产出
- 输出规格
- 所属系列 / 产品
- 合成进度
- 状态
- 提交时间
- 操作

其中：
- 合成包行需展示：包名称、策略名称、输入物料数量、目标产出数量、输出比例/时长/编码规格。
- 子结果行需展示：结果文件名称、输入摘要、输出成片规格。

### 5.3 状态与操作
#### 合成包状态
- `排队中` / `合成中`：操作为 `终止任务`、`展开/收起`
- `部分失败`：操作为 `重试失败项`、`查看详情`、`展开/收起`
- `已完成`：操作为 `查看结果`、`导出结果`、`展开/收起`
- `已终止`：操作为 `查看详情`、`展开/收起`

#### 子结果状态
- `合成成功`：操作为 `查看详情`
- `合成失败`：操作为 `重试`、`查看日志`
- `合成中`：操作为 `查看详情`

## 6. 展示与交互规范
- 状态必须使用带底色与文字的标签，不使用无意义圆点。
- 进度统一采用“数值 + 进度条”组合展示。
- 包任务与子任务需通过背景层级和缩进关系做出区分。
- 操作区采用文本按钮形式，危险操作使用错误色标识。
- 列表需支持横向滚动，保证字段完整性优先于强行压缩。

## 7. 页面跳转关系
- 从“素材上传”“物料上传”等入口发起上传后，用户可在 `我的任务 > 上传` 中查看任务详情。
- 从“素材库”等入口触发下载后，用户可在 `我的任务 > 下载` 中查看任务详情。
- 从“合成策略”页点击 `正在合成的任务` 后，应直接打开本页并默认切换到 `视频合成` 视图。
