# 素材系统总需求文档

## 一、项目概述

### 1.1 项目背景
- 当前量化投放业务同时存在素材、物料、合成三类核心资产，但页面职责、流程口径、状态口径尚未统一。
- 素材上传、物料上传、素材下载、视频合成等异步处理分散在不同页面和动作中，用户难以形成完整的任务追踪闭环。
- 合成策略、新建策略、素材详情等页面虽然已有独立需求，但缺少一份面向整个素材系统的统一需求文档，导致字段、状态、跳转关系容易出现不一致。
- 现阶段需要先统一产品层定义，为后续原型、页面开发、接口设计和联调提供一致基线。

### 1.2 项目目标
构建一套围绕素材生产与管理的后台系统，让业务、产品、设计、研发能够基于同一套定义协同推进：

1. 统一管理素材与物料两类资产。
2. 建立手动上传素材/物料、检测、审核、入库的标准流程。
3. 建立下载素材/物料、视频合成等异步处理的统一任务中心。
4. 支持合成策略的创建、编辑、启停与结果追踪。
5. 支持查看单个素材的完整业务信息。
6. 统一页面范围、字段口径、状态流转、按钮动作与页面跳转关系。

### 1.3 核心设计原则

| 原则 | 说明 |
|---|---|
| 资产分层明确 | 素材与物料不是同一对象；素材用于投放，物料用于合成素材。 |
| 任务统一归集 | 上传、下载、视频合成三类异步任务统一沉淀到“我的任务”。 |
| 页面职责单一 | 固定菜单页承载工作台能力，非菜单页承载详情、上传、策略配置等业务页能力。 |
| 状态统一管理 | 上传、下载、视频合成、策略状态必须在系统内保持统一定义和统一展示。 |
| 结果回流工作台 | 上传完成的素材回流素材库，上传完成的物料回流物料库，合成结果回流任务中心。 |

## 二、系统架构

### 2.1 页面层级结构
系统由 4 个固定菜单页面和 4 个非固定菜单业务页面组成，层级关系如下：

```text
素材管理（固定菜单）
├── 素材库              ← 入口页，管理所有素材
│   ├── 素材详情        ← 单个素材的完整信息页
│   └── 上传素材        ← 素材上传与入库前处理页
├── 物料库              ← 入口页，管理所有物料
│   └── 上传物料        ← 物料上传与入库前处理页
├── 合成策略            ← 管理所有合成策略
│   └── 新建策略        ← 单条策略的新建与编辑页
└── 我的任务            ← 上传/下载/视频合成统一任务中心
```

### 2.2 数据模型概览
素材库、物料库的详细数据模型以飞书字段表为准：
https://q3t5rgpff5.feishu.cn/sheets/MmOHsH6ZKh5aMmt4xuVcJfmLnUd?from=from_copylink

```text
Asset（素材）
├── 基础标识信息
├── 文件信息
├── 标签信息
├── 生产协作信息
├── 状态信息
├── 媒体信息
└── 效果信息

Material（物料）
├── 基础标识信息
├── 文件信息
├── 标签信息（含位置类信息）
├── 生产协作信息
└── 状态信息

Strategy（合成策略）
├── 基础信息

UploadTask（上传任务）
├── 文件信息
├── 归属信息
├── 进度信息
└── 状态信息

DownloadTask（下载任务）
├── 文件信息
├── 归属信息
├── 进度信息
└── 状态信息

ComposeTask（视频合成任务）
├── 合成包信息
├── 输入/输出信息
├── 进度信息
└── 状态信息
```

### 2.3 关键数据流
素材、物料、策略与任务之间的数据流如下：

```text
素材库 ──上传素材──→ 上传素材页 ──创建上传任务──→ 我的任务（上传）
   ↑                                                  │
   └────────────── 上传完成并入库后回流 ───────────────┘

物料库 ──上传物料──→ 上传物料页 ──创建上传任务──→ 我的任务（上传）
   ↑                                                  │
   └────────────── 上传完成并入库后回流 ───────────────┘

素材库/物料库 ──发起下载──→ 我的任务（下载）

合成策略 ──新建/编辑──→ 新建策略页 ──保存策略──→ 合成策略
   │
   └────────────── 生成视频合成任务 ───────────────→ 我的任务（视频合成）

素材库 ──查看详情──→ 素材详情页
```

核心约束：工作台页负责发起动作，任务中心负责承接状态与结果。

## 三、页面详细设计

### 3.1 素材库页
文件：`素材管理/素材库.html`  
入口：左侧导航「素材库」  
职责：展示所有素材资产，支持筛选、目录查看、详情查看、发起上传、发起下载及批量处理。

#### 3.1.1 数据字段
素材库详细字段以飞书字段表为准：
https://q3t5rgpff5.feishu.cn/sheets/MmOHsH6ZKh5aMmt4xuVcJfmLnUd?from=from_copylink

本页在产品层重点承接以下必要字段，详细字段定义仍以飞书字段表为准：
- 筛选字段：素材名称、查看范围、媒体、状态、素材类型、项目、设计师、类型标签、内容标签、优化师、创建时间、使用数据。
- 目录字段：文件夹搜索、目录树层级。
- 卡片字段：缩略图、素材名称、状态角标、素材类型、时长、分辨率、标签摘要。
- 上下文字段：进入素材详情所需的素材主标识、当前目录、当前筛选条件。

#### 3.1.2 功能点
- 素材库负责承接素材检索、目录查看、上传入口、下载入口和批量处理，是素材资产的统一工作台。
- 素材库不负责承载单个素材的完整信息查看，单个素材详情由素材详情页独立承接。
- 素材库发起的上传、下载等异步动作统一沉淀到“我的任务”，当前页只负责发起，不负责展示任务过程。
- 素材库中的批量动作只定义业务入口与结果归属，不扩展任务状态机细节。

#### 3.1.3 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 查询 | 按筛选条件检索素材 | 刷新素材列表 |
| 清空 | 清空筛选条件 | 清空当前筛选项并恢复默认查询状态 |
| 展开/收起 | 展开或折叠扩展筛选项 | 显示或隐藏扩展筛选字段 |
| 上传素材 | 打开上传素材页 | 进入 `上传素材.html` |
| 选择 | 进入多选模式 | 卡片进入可批量选择状态 |
| 全选/取消全选 | 批量选择当前结果 | 选中或取消选中当前结果集 |
| 移动 | 批量移动素材 | 打开目标路径选择并完成移动 |
| 收藏 | 批量收藏素材 | 将选中素材加入收藏 |
| 转移到待优化库 | 批量转移素材 | 将选中素材转移到待优化库 |
| 删除 | 批量删除素材 | 删除前需二次确认 |
| 下载 | 下载单个素材 | 创建下载任务并进入任务中心追踪 |
| 编辑 | 查看素材详情 | 打开 `素材详情.html` |
| 入库 | 执行单卡入库动作 | 按当前素材状态进入对应入库处理 |
| 移动（单卡） | 移动当前素材 | 打开目标路径选择并完成移动 |
| 收藏（单卡） | 收藏当前素材 | 将当前素材加入收藏 |

### 3.2 物料库页

本页在产品层重点承接以下必要字段，详细字段定义仍以飞书字段表为准：

- 物料库不单独承接复杂的任务过程展示，上传与下载后的过程统一沉淀到“我的任务”。
- 物料库的核心业务价值在于为新建策略页提供可筛选、可预览、可组合的输入资源池。

### 3.3 合成策略页
文件：`素材管理/合成策略.html`  
入口：左侧导航「合成策略」  
职责：展示所有合成策略，支持筛选、状态管理、批量管理，并作为新建策略页和视频合成任务的入口。

#### 3.3.1 数据字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 系列 | string | 按系列筛选策略 |
| 产品 | string | 按产品筛选策略 |
| 状态 | enum | 全部 / 生效中 / 已暂停 |
| 搜索词 | string | 按策略名称或策略 ID 搜索 |
| 策略名称 | string | 卡片主标题 |
| 策略状态 | enum | 生效中 / 已暂停 |
| 业务标签 | string[] | 系列、产品及附加标签 |
| 累计消耗 | number | 展示策略消耗 |
| 合成素材数 | number | 展示策略产出量 |
| 操作人 | string | 最近操作人 |
| 操作时间 | datetime | 最近操作时间 |
| 策略编号 | string | 策略唯一标识 |

#### 3.3.2 状态机制
策略状态只保留两种：
- 生效中：策略可继续产生新的视频合成任务。
- 已暂停：策略暂停，不再继续产生新任务。

状态流转逻辑如下：

```text
生效中 → 已暂停
已暂停 → 生效中
```

#### 3.3.3 功能点
- 合成策略页负责承接策略查询、策略状态管理、策略批量管理和任务入口管理，是策略对象的统一工作台。
- 合成策略页不承接具体策略配置细节，单条策略的配置由新建策略页独立承接。
- 合成策略本身是规则对象，不等于视频合成任务对象；任务执行过程统一沉淀到“我的任务-视频合成”。
- 本页重点解决“有哪些策略、当前是否生效、由谁操作、产出规模如何、如何进入配置页和任务页”这几个问题。

### 3.4 我的任务页
文件：`素材管理/我的任务.html`  
入口：左侧导航「我的任务」；或由上传、下载、合成策略等页面跳转进入  
职责：集中查看手动上传素材/物料、下载素材/物料、视频合成三类异步任务，是素材管理模块的统一任务中心。

#### 3.4.1 页面分区
页面由四个一级任务视图组成：

| 区域 | 说明 |
|---|---|
| 待操作 | 聚合需要人工处理的异常或待办任务（上传失败、同质化过高、合成失败、待审核） |
| 上传 | 查看素材/物料上传任务，支持单文件任务与上传包任务 |
| 下载 | 查看下载任务 |
| 视频合成 | 查看视频合成任务与合成包结果 |

#### 3.4.2 数据字段

**公共字段**

| 字段 | 类型 | 说明 |
|---|---|---|
| 一级任务类型 | enum | 待操作 / 上传 / 下载 / 视频合成 |
| 时间范围 | daterange | 按提交时间筛选 |
| 文件名称/任务名称 | string | 按名称检索 |
| 所属系列/产品 | string | 按业务归属筛选 |
| 进度 | number | 任务当前进度 |
| 状态 | enum | 当前任务状态 |

**待操作字段**

| 任务/文件名称 | string | 展示任务或文件名称 |
| 所属系列/产品 | string | 展示业务归属 |

**上传/下载字段**

|---|---|---|

#### 3.4.3 状态流转

**上传任务状态流转**

```text
待执行 → 上传中 → 上传成功 → 同质化检测中 → 待审核 → 待入库 → 入库中 → 已完成
上传中 → 上传失败
同质化检测中 → 检测未通过
待审核 → 审核不通过
任意允许删除的状态 → 已删除
上传失败 → 重新开始 → 上传中
```

**下载任务状态流转**

```text
下载中 → 下载成功
下载中 → 下载失败
下载失败 → 重新下载 → 下载中
```

**视频合成任务状态流转**

```text
等待中 → 同质化检测中 → 同质化合格 → 上传中 → 上传成功
等待中 / 同质化检测中 / 上传中 → 已暂停
已暂停 → 开始/恢复 → 原执行阶段
同质化检测中 → 同质化过高
上传中 → 上传失败
上传失败 → 重试 → 上传中
等待中 / 同质化过高 / 上传失败 / 已暂停 → 已删除
```

#### 3.4.4 功能点
- 我的任务页负责承接上传、下载、视频合成三类异步任务的统一查看，不负责发起上传、下载或策略配置动作。
- 三类任务在同一页面内以独立视图承接，各自保留筛选条件、字段定义和状态机口径。
- 上传包与合成包可在当前页展开查看子项，但任务执行结果仍以对应状态机为准，不额外引入独立状态口径。
- 本页重点解决“任务现在处于什么状态、还能做什么、结果在哪里看”这三个问题。

#### 3.4.5 操作按钮

**上传任务相关按钮**

| 按钮 | 说明 | 结果 |
|---|---|---|
| 上传 Tab | 切换到上传视图 | 展示上传任务 |
| 重置筛选 | 清空当前上传视图筛选条件 | 清空当前上传视图筛选项并恢复默认状态 |
| 暂停 | 暂停当前上传任务 | 将任务状态更新为已暂停（仅适用于允许暂停的状态） |

### 3.5 素材详情页
文件：`素材管理/素材详情.html`  
入口：素材库 → 点击素材卡片主体或编辑按钮  
职责：查看单个素材的完整业务信息，并作为素材库进入单素材深度查看的唯一详情页。

#### 3.5.1 页面结构
素材详情页的信息组织遵循“先看素材本身，再看核心数据，最后看媒体差异”的顺序：
- 第一层为素材预览与基础信息，用于让用户先确认当前看的是什么素材。
- 第二层为数据总览，用于快速判断该素材是否值得继续关注。
- 第三层为媒体信息，用于区分同一素材在不同媒体平台上的审核与表现差异。
- 低频或补充字段放在其他补充信息区，不抢占首屏信息密度。

#### 3.5.2 数据字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 页面标题 | string | 固定为“素材详情” |
| 素材预览 | media | 图片或视频内容 |
| 基础信息 | object | 名称、类型、尺寸、时长、标签、生产信息等 |
| 数据总览 | object | 消耗、展示、点击、ROI 等核心指标 |
| 媒体信息 | MediaInfo[] | 按平台查看审核、表现、评论等信息 |
| 其他补充信息 | object | 低频字段补充区 |

#### 3.5.3 功能点
- 素材详情页负责承接单个素材的完整信息查看，不承担素材列表检索、批量处理或任务过程展示。
- 本页的核心价值是把素材预览、业务信息和媒体表现放到同一页内，避免用户在素材库与其他页面之间频繁来回跳转。
- 关闭详情页后，用户应返回素材库原上下文，不改变原筛选、排序和分页状态。

#### 3.5.3 操作按钮

| 按钮 | 说明 | 结果 |
|---|---|---|
| 关闭 | 关闭当前详情页 | 返回素材库上下文 |
| 编辑 | 进入编辑动作 | 打开当前素材对应的编辑或处理入口 |
| 平台切换 Tab | 切换媒体平台 | 刷新媒体信息区 |
| 展开更多 | 查看低频信息 | 展开补充字段 |

### 4.2 技术栈
当前页面默认采用单文件 HTML 进行原型与页面承载，运行方式依赖系统框架以 iframe 方式加载。

- HTML：页面结构与业务内容承载。
- CSS：页面样式与布局控制。
- JavaScript：页面内状态、按钮动作、页面间 `postMessage` 通信。
- 菜单配置：通过 `menu-config.js` 维护固定菜单与页面映射。
- 框架加载：通过系统主框架 `index.html` 统一承载页面、菜单、标签页与跳转。

### 4.3 统一设计规范

| 规范项 | 值 |
|---|---|
| 页面背景 | `#f5f7fa` |
| 页面内边距 | `8px` |
| 页面定位 | 运行于 iframe 内，不单独承担导航框架 |
| 新页面技术默认 | 原生 HTML / CSS / JavaScript |
| 页面高度规则 | 不使用 `100vh`，由内容自然撑开 |
| 定位规则 | 不使用 `position: fixed` 以避免遮挡框架 |
| 跳转规则 | 通过框架页内动作打开业务页，不直接替换当前框架 |
| 菜单规则 | 固定菜单仅保留素材库、物料库、合成策略、我的任务 |
