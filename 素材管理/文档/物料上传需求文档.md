# 物料上传需求文档

## 一、页面结构
整体结构与“素材上传”页面保持统一，采用左 60% 管理区、右 40% 配置区的分栏布局，页面外边距与内部卡片间隙严格遵循系统 8px UI 规范。在核心结构不变的基础上，针对**物料**的特性（如位置属性、动态扩展类别）在标签管理模块进行了深度定制化交互设计。

## 二、左侧：上传与物料管理区

### 1. 目标文件夹更换
- 提供可视化当前路径（如：`我的物料 / 测试`）。
- 支持点击更换，呼出树状目录结构选择新的挂载物料库节点。

### 2. 文件上传组件
- 极简压缩态上传框（固定高度 120px 左右），保证下方物料列表的展示空间。
- **支持格式**：`mp4`, `mov`, `avi`, `png`, `jpg`, `jpeg`, `gif`
- **体积限制**：单文件 ≤ 500MB，单次批量总计 ≤ 4096MB。
- **上传主流程调整**：物料文件选定后先执行上传，上传成功后再进入同质化检测，不再在上传前做同质化检测。
- **上传后处理顺序**：主路径统一为 `上传中 → 上传成功 → 同质化检测中 → 待审核 → 待入库 → 入库中 → 已完成`。
- **用户确认节点**：入库前必须由用户显式确认；只有在审核通过后，任务才进入“待入库”，用户点击“入库”后才真正开始入库。
- **异常分支**：上传失败支持重试；同质化检测未通过、审核不通过、入库失败均需保留对应状态，并支持删除或在允许的节点上重试。

### 3. 物料列表与交互操作
- **基本操作**：卡片化展示物料缩略图、分辨率与时长；支持多选/全选批量控制，右上角支持快捷删除。
- **快速重命名**：物料标题支持“内联编辑”（Inline Edit）。点击名称右侧✏️即可原地唤出输入框，修改完成后敲击回车或点击“✓”保存到本地状态。
- **AI打标交互**：提供单独的 AI 处理按钮。用户点击并确认覆盖风险后：
  1. 弹出顶部 Toast：“已开始上传，您可在「我的任务 > 上传」中查看详情”；
  2. 按钮立刻进入 loading 状态（“打标中”），防止二次点击；
  3. 左侧紧邻的“编辑标签”按钮将被禁用（disabled），防止 AI 处理期间发生人工篡改冲突。
- **任务状态同步**：`我的任务 > 上传` 中的物料任务状态需覆盖：`待执行`、`上传中`、`上传成功`、`同质化检测中`、`检测未通过`、`待审核`、`审核不通过`、`待入库`、`入库中`、`入库失败`、`已完成`、`已删除`。
- **检测与入库反馈**：检测完成后需在任务详情中展示检测结果；审核通过后给出入库提醒，但不自动入库，需由用户手动点击“入库”。

## 三、右侧：信息配置区
- **按需加载的人员字段**：两列布局，默认显示高频字段「设计师」和「剪辑师」；其他关联角色（如物料供应商、编导、演员等）统一收纳在「展开更多」折叠面板中。
- **人员录入方式**：所有角色字段需支持混合录入（即允许在现存下拉菜单中选择，也允许用户自行敲字新建人员名称）。
- **备注描述**：提供多行文本框以便录入物料要求说明。

## 四、物料专有标签展示与弹窗设计（特化交互）

### 1. 外部卡片标签展示
- **顺序与颜色**：严格按照 `[ 位置 (紫色) ] -> [ 类型 (蓝色) ] -> [ 内容 (灰色) ] -> [ 定向 (橙色) ]` 的语义化结构排列。
- **动态截断机制**：单张卡片最多显式展示 4 个标签，超过的部分折叠为 `+X` 标签，鼠标悬停时通过气泡展示全部内容。

### 2. 弹窗整体布局（多栏顺序工作流）
点击“编辑标签”呼出 1200px 超宽弹窗，重构为符合直觉的顺序配置结构：
- **分布式搜索与 AI**：摒弃全局搜索。类型、内容、定向标签栏**内部分别置顶专属搜索框**；每个栏目标题右侧独立设置 **“AI识别”** 按钮（点击后独立 loading 并自动勾选）。

### 3. 应用位置（条件单选）
- **条件判断**：系统自动识别文件类型，**若为图片类型则自动隐藏本栏**，其余栏目动态向左侧平移填补。
- **交互形式**：采用竖向排列单选，去除“无限制”冗余选项，预设 `[视频开头] [视频中间] [视频结尾]`，默认初始为空。

### 4. 树状层级标签（悬停拓展机制）
适用于“类型标签（单选）”、“内容标签（多选）”与“定向标签（多选）”：
- **精准下级拓充**：抛弃全局新增框。当鼠标悬停于任意具有下属目录的“文件夹节点”上时，浮现 `+` 图标（底层叶子节点不可新增）。
- **新增录入**：点击 `+` 唤出子级录入弹窗，确认后新建节点挂载于该父级目录下并被即时自动选中。

### 5. 组合展示与统一已选池
为解决海量多选带来的状态迷失，将涉及多选的模块进行整合：
- **树状选择区并列**：内容标签树与新加入的**定向标签**树（包含人群、地域、兴趣等）紧凑并列。
- **统一视图池（Selected Pool）**：在右侧划出独立区域收敛所有已选结果。已选内容与定向标签分组排列，用户可在此直观总览，点击 `×` 取消时，左侧树状态将完美双向同步。
