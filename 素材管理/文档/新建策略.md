# 新建策略 (add-strategy) 需求文档

## 1. 业务背景
当前量化投放系统需要一个专门的“策略配置”页面，用于定义短视频或图片物料的自动化合成规则。用户通过编排合成节点（片头、中间部分、结尾等），并为每个节点配置特定的筛选条件和标签，最终生成完整的投放策略方案。

> 本页文件路径为 `素材管理/新建策略.html`，并通过 `素材管理/合成策略.html` 页顶部的 `+ 新增策略` 按钮在系统框架内以新标签页方式打开；后续同一页面也承担策略编辑态。

## 2. 页面结构与布局
页面采用左右分栏布局（Flex 结构，固定高度 100vh，无页面级滚动条）：

*   **左侧：合成策略编排流（Sidebar）**
    *   作用：定义合成的逻辑顺序与步骤。
    *   宽度：固定宽度（280px）。
*   **右侧：策略参数配置区（Main Panel）**
    *   作用：设置全局参数以及当前选中节点的具体配置，并提供符合条件的素材预览。
    *   宽度：自适应剩余空间（Flex: 1）。

## 3. 功能模块详细说明

### 3.1 左侧：策略节点编排 (Flow Editor)
*   **节点类型**：
    *   **固定节点**：“片头”、“结尾”。不可删除。
    *   **动态节点**：“中间1”、“中间2”等。允许用户新增、删除或调整顺序。
*   **节点状态**：
    *   **默认态**：灰底/白底，极细边框。
    *   **激活态 (Active)**：边框高亮（主色），带有明显阴影，并且节点右侧有一个指向右侧配置面板的三角形箭头（指示关联关系）。
    *   **悬停态 (Hover)**：微小的位移和阴影增强反馈。
*   **节点操作**：
    *   左侧常驻拖拽把手图标（暗示可排序）。
    *   右侧常驻操作图标（动态节点显示“删除垃圾桶”，已配置完成节点显示“绿色对勾”）。
*   **连接线与新增**：
    *   节点之间通过垂直线段连接。
    *   连接线中间悬浮一个“+”号按钮，用于在此处插入新节点。

### 3.2 右侧顶部：全局策略设置 (Global Config)
*   **折叠栏设计**：为了节省垂直空间，该模块默认可折叠。
*   **收起态 (Summary)**：展示核心信息的摘要（策略名称、关联项目、投放定向、目标生成数量、入库审核状态）。
*   **展开态 (Form)**：展示详细配置表单，包含：
    *   **策略名称**（输入框）
    *   **关联项目**（单选：通过项目选择器触发，仅保留最新选中的一个项目）
    *   **投放定向**（多选：通过专门的定向选择弹窗触发，展示选中的标签列表）
    *   **目标生成数量**（数字输入框：提示“不输入为不限制数量”）
    *   **入库是否需要审核**（单选：是/否）

### 3.3 右侧中部：当前节点配置 (Node Config)
*   **标题联动**：标题“节点配置：xxx”需与左侧当前激活的节点名称保持同步。
*   **节点级筛选范围调整**：项目范围与投放定向相关配置统一以上方全局策略设置为准。
*   **标签筛选分拆 (Tag Categories)**：
    *   字段拆分为：**卖点、钩子、表现形式、落版**。
    *   UI 设计：采用 4 列网格布局，每个字段均拥有独立的标签筛选区域（Tag Area）。
    *   核心逻辑：每个分类内部满足任一标签即可匹配（OR 逻辑），分类之间共同作用约束素材范围。
    *   交互：每个区域均支持点击“编辑”按钮调起标签选择器，并支持胶囊标签的点击删除。
*   **定向筛选区域 (Tag Area)**：
    *   展示位置：位于“标签筛选”下方，作为独立筛选区块展示。
    *   需求含义：用于配置节点级定向条件标签，便于在全局定向基础上进一步约束当前节点的可选素材范围。
    *   交互逻辑：与标签筛选保持一致，支持标签盒展示、编辑与删除。

### 3.4 右侧底部：物料预览 (Material Explorer)
*   **设计参考**：高度复刻系统的“素材库”功能。
*   **顶部工具栏**：
    *   左侧：数据概览（展示符合当前筛选条件的素材总数）。
    *   右侧：批量操作入口（“一键选中所有结果”、“确认选择”等）。
*   **左侧分类树 (Directory Tree)**：
    *   分为“全部物料”、“2026 夏季新品”、“历史存档”、“未分组资源”等。
    *   支持目录层级高亮展示。
*   **右侧素材网格 (Asset Grid)**：
    *   **网格工具栏**：包含全局搜索框、**排序选项**（按消耗倒序、按消耗正序）。已移除冗余的媒体类型、状态及视频/图片筛选。
    *   **素材卡片 (Asset Card)**：采用“上图下文”的紧凑型设计。
        *   **缩略图**：16:9 比例，右上角带有状态角标（如：未投放-灰，无消耗-黄，有消耗-蓝）。
        *   **基础信息**：两行截断的素材名称。
        *   **元数据**：底部展示类型（图标）、时长/属性、分辨率等。
        *   **禁用/恢复交互**：卡片右下角提供状态切换按钮。
            *   点击“禁用”：弹出二次确认窗（“请问要禁用该物料吗”），确认后卡片变为灰色半透明状态。
            *   点击“恢复”：弹出二次确认窗（“请问要恢复该物料吗”），确认后卡片恢复正常状态。
        *   **预览交互**：点击卡片非按钮区域可唤起媒体预览（视频支持自动播放）。
    *   **分页器**：底部常驻分页控件。

## 4. 交互核心要求 (UX & UI)
*   **去线框化**：大量使用阴影（Shadow）区分层级，而非粗糙的实线边框，以提升产品的高级感。
*   **空间利用率**：通过折叠“全局设置”、采用流式“标签盒”以及压缩“素材卡片”的间距，将超过 50% 的页面高度留给核心的“素材网格”区域。
*   **视觉连贯性**：确保左侧流与右侧表单的当前操作具有强烈的视觉绑定感（例如左侧的 Active 箭头直接指向右侧表单区域）。

## 5. 待优化方向（记录）
*   左侧编排区域的图标比例、线条粗细以及状态反馈需进一步精修（如调整为虚线连接、弱化加号按钮）。
*   多选与移动交互（如批量入库、收藏等）。
