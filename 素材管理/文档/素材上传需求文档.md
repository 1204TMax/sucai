# 素材上传需求文档

## 一、页面结构
整体采用左右两列布局的弹窗/抽屉视角模式，左侧核心管理区占比 60%，右侧信息配置区占比 40%。页面外层边距必须符合全局框架的 `8px` 间距规范。

## 二、左侧：上传与素材管理区

### 1. 上传目录选择
- 顶部展示当前选中的目标文件夹路径（例：`我的素材 / 素材库 / 测试`）。
- 点击「更换文件夹」弹出选择器，以树状结构 (`Tree`) 展示全部可用目录，选中后动态刷新当前路径。

### 2. 拖拽上传区
- 采用“紧凑型”拖拽框设计（高度强制约束在约 120px），包含虚线边框与图标。
- **支持文件格式**：`mp4`, `mov`, `avi`, `png`, `jpg`, `jpeg`, `gif`
- **尺寸与容量限制**：单个文件 ≤ 500MB；当次批量上传总大小 ≤ 4096MB。
- **上传主流程调整**：文件选定后先执行上传，上传成功后再进入同质化检测，不再在上传前做同质化检测。
- **上传后处理顺序**：主路径统一为 `上传中 → 上传成功 → 同质化检测中 → 待审核 → 待入库 → 入库中 → 已完成`。
- **用户确认节点**：入库前必须由用户显式确认；只有在审核通过后，任务才进入“待入库”，用户点击“入库”后才真正开始入库。
- **异常分支**：上传失败支持重试；同质化检测未通过、审核不通过、入库失败均需保留对应状态，并支持删除或在允许的节点上重试。

### 3. 素材列表与卡片管理
- **全选控制**：支持通过顶部的 Checkbox 一键全选/取消全选已上传的卡片。
- **素材卡片结构**：
  - **左侧**：缩略图，底部悬浮展示黑色半透明的分辨率（如 1080×1920）与视频时长（如 00:41）。
  - **中部（信息与重命名）**：展示素材名称。提供内联重命名（Inline Edit）功能，点击名称旁边的「编辑图标」，文本原地转变为输入框，提供【✓确认】与【×取消】按钮，或回车保存。
  - **中部（标签展示）**：每个卡片最多外显展示 2 个标签。多于 2 个的部分收起为 `+N` 的样式，鼠标悬停时（Hover）气泡展示全部标签。
  - **卡片操作区**：
    - 「编辑标签」按钮：点击呼出标签配置大弹窗。
    - 「AI打标」按钮：点击触发覆盖提醒弹窗。用户确认后弹出 Toast（“已开始上传，您可在「我的任务 > 上传」中查看详情”），且该按钮变为 loading（“打标中”）状态，同时禁用左侧的“编辑标签”以防冲突。
    - 上传完成后，任务统一沉淀到 `素材管理/我的任务.html` 的「上传」视图中查看；列表需同时兼容**单文件任务**与**上传包任务**。上传包用于承载一次批量上传中的多个文件，并支持在任务列表中展开/收起查看子文件进度。
    - 「我的任务 > 上传」中的任务状态需覆盖：`待执行`、`上传中`、`上传成功`、`同质化检测中`、`检测未通过`、`待审核`、`审核不通过`、`待入库`、`入库中`、`入库失败`、`已完成`、`已删除`。
    - 当检测完成后，任务详情中需展示检测结果；审核通过后给出入库提醒，但不自动入库，需由用户手动点击“入库”。
    - 右上角提供「删除 (X)」快捷键。

## 三、右侧：信息配置区

### 1. 人员信息模块
- 采用两列 Grid 布局排版。
- **默认折叠机制**：默认只显示最核心的「设计师」与「剪辑师」两个字段。
- **展开更多**：点击右侧操作项后，无缝展开并展示：素材供应商、优化供应商、编导、策划、拍摄、演员。
- **输入交互**：所有人员字段使用 `Select mode="tags"` 组件。既支持在下拉列表中点选已有人员名单，也支持直接用键盘输入自定义的人员名字并回车录入。

### 2. 备注模块
- 底部提供 `TextArea` 多行文本框，供业务填写选填信息。

## 四、标签编辑大弹窗

采用宽版 Modal，分为顶部搜索、左列单选树、右列多选树三个区域。

### 1. 顶部操作
- 包含带搜索图标的 `Input`，支持对树内标签的快速检索筛选。

### 2. 类型标签（左侧区）
- 代表该素材所属的物理/内容体裁。
- 使用 `Tree` 组件呈现并设为**单选模式**（例如：海报 -> 节日海报）。

### 3. 内容与自动标签（右侧区）
- **内容标签**：代表素材的内在属性。使用 `Tree` 组件呈现并设为**复选模式**（支持同时勾选多个底层叶子节点）。
- **系统自动识别标签**：在下方以专属列表呈现 AI 模型自动打上的标签（带有“自动:”前缀标识）。提供关闭 `closable` 属性，允许人工移除识别错误的条目。
