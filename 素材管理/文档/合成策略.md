# 需求文档：合成策略

## 1. 页面基本信息
- **页面名称**：合成策略
- **所属模块**：素材管理
- **文件路径**：`素材管理/合成策略.html`
- **主要目标**：供企业管理员管理自动化素材生产的规则与策略，支持策略的检索、状态监控、数据回收以及批量操作。

## 2. 页面布局与结构
页面整体采用**“模块化白盒”**与**“区块化”**设计，划分为以下核心区域：

### 2.1 顶部控制中枢 (区块 A)
- **页面标题**：展示“合成策略”及当前记录总数（如“共 48 条记录”）。
- **全局操作**：
  - `正在合成的任务`（次要按钮）：点击后在框架内打开 `素材管理/我的任务.html` 标签页，并默认聚焦到「视频合成」视图。
    - 该页面不是 AI 创作模块中的“我的任务”，而是**素材管理模块专属的任务中心**，用于统一查看上传、下载与视频合成三类任务。
    - 「视频合成」任务状态需按状态机展示，包含：`等待中`、`已暂停`、`同质化检测中`、`同质化合格`、`同质化过高`、`上传中`、`上传成功`、`上传失败`、`已删除`。
    - 视频合成任务主路径为 `等待中 → 同质化检测中 → 同质化合格 → 上传中 → 上传成功`；其中 `同质化过高` 作为检测分支保留展示。
    - `等待中`、`同质化检测中`、`上传中` 三个阶段支持“暂停”，暂停后进入 `已暂停`；在 `已暂停` 状态下支持“开始”恢复到对应执行阶段。
    - `上传失败` 支持“重试”回到 `上传中`；`等待中`、`同质化过高`、`上传失败`、`已暂停` 均支持删除并流转到 `已删除`。
  - `+ 新增策略`（主按钮）：用于创建新的合成规则。点击后需在框架内打开 `素材管理/新建策略.html` 标签页，进入策略新建页面。
- **筛选容器**：
  - 下拉筛选：支持按“全系列”、“全部产品”、“状态（全部/生效中/已暂停等）”筛选。
  - 搜索框：支持通过“策略名称”或“策略 ID”进行精准模糊匹配。
  - 动作按钮：包含 `查询`（主色按钮）和 `重置`（次要按钮）。

### 2.2 列表与管理工具栏 (区块 B)
- **排序组件**：位于列表右上方，支持以下维度排序：
  - 创建时间：由近及远 / 由远及近
  - 合成数量：由多到少 / 由少到多
  - 累计消耗：由高到低 / 由低到高
- **管理模式（交互核心）**：
  - **默认态**：隐藏复选框，列表呈现纯净的数据展示模式。提供一个 `选择` 按钮。
  - **管理态**：点击 `选择` 后，进入批量管理模式：
    - 列表所有卡片左侧滑出复选框。
    - 工具栏出现 `全选` 勾选框。
    - 出现带有下拉菜单的 `批量操作` 按钮（包含：批量暂停、批量启动、批量删除）。其中“批量删除”为危险操作（标红）。
    - 出现 `退出选择` 按钮，点击恢复默认态并清空所有选中状态。
  - **二次确认**：点击任何批量操作项，必须拉起系统级确认弹窗（Confirm Modal），提示当前选中的数量并请求二次确认。

### 2.3 策略数据卡片 (核心内容承载)
卡片采用**“三段式横向流”**布局，强调信息层级与视觉对齐：

1. **左侧：身份识别区**
   - **复选框**：仅在管理模式下可见。
   - **主标题**：策略名称（15px 粗体，超长截断）。
   - **状态标签**：紧跟标题（如：生效中、已暂停），采用带底色的胶囊样式。
   - **业务标签**：位于下方，必定包含策略所属的“系列”和“产品”（如：减脂系列、享瘦产品），其余为平台或格式等附加标签，采用灰色底色小标签。
   
2. **中间：指标看板区**
   - 包含 `累计消耗 (元)` 和 `合成素材 (个)` 两个核心指标。
   - 布局：与左侧标题/标签平齐，左侧辅以 1px 垂直分割线进行物理隔离。
   - 视觉排版：Label 在上（10px 灰色），Value 在下（16px 加粗），营造仪表盘专业感。

3. **右侧：操作与溯源区**
   - 通过弹性占位符（Spacer）与中间指标区物理隔开，确保靠右对齐且不换行。
   - **操作组 (Top)**：
     - **基础操作**：
       - `查看表现`：跳转至数据表现页，查看该策略下的素材跑量与转化数据（灰黑色幽灵按钮）。
       - `编辑`：点击后需在框架内打开 `素材管理/新建策略.html` 标签页，并以编辑态加载当前策略已有配置，允许用户修改生产规则与定向等参数（灰黑色幽灵按钮）。
     - **状态控制操作**：固定宽度（64px）的蓝色主色按钮，按钮文案与左侧的【状态标签】互斥对应：
       - 当状态为 **`生效中`** 时，显示 **`暂停`** 按钮。点击后阻断该策略的后续生产任务，并将状态更新为“已暂停”。
       - 当状态为 **`已暂停`** 时，显示 **`启动`** 按钮。点击后恢复该策略的生产任务，并将状态更新为“生效中”。
   - **元数据/溯源 (Bottom)**：
     - 展示 `操作人 | 操作时间 | 策略编号`。
     - 采用浅灰色小字（11px），作为卡片的底层背景信息。

## 3. 依赖说明
- **样式体系**：依赖 `共享样式/components.css` 中的 Design Tokens（颜色、间距、圆角等全局变量）。
- **组件库**：依赖 `共享样式/confirm-modal.css` 及 `共享样式/confirm-modal.js` 提供标准化的二次确认弹窗能力。
