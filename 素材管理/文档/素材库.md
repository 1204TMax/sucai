# 素材库需求文档 (PRD)

## 1. 页面概述
“素材库”页面是用于管理、检索、分类和操作所有素材资产的核心工作台。它采用左侧分类树 + 右侧素材网格的经典双栏布局，并针对屏幕空间的利用率和操作的便捷性进行了高度精细化设计。

除列表浏览外，页面还需提供一个由素材卡片进入的“素材详情”查看视图。当前该视图已作为独立页面落地为 `素材管理/素材详情.html`，并通过系统框架以新标签页方式打开；原型来源文件为 `素材管理/原型/素材详情.html`，用于承载素材预览、基础信息、数据总览和媒体信息等高密度内容。

## 2. 界面布局与排版规范
*   **外层边距**：页面主体及各大模块之间统一采用 `8px` 间距。
*   **基础控件**：所有按钮、下拉框、输入框高度降至 `32px`，按钮内边距调整为 `6px 12px`，统一字号为 `13px`。
*   **卡片网格**：素材卡片容器采用 Grid 布局，最小卡片宽度缩小为 `180px`，卡片之间的间隙 (gap) 为 `12px`。
*   **长文本处理**：素材名称固定支持最多2行展示（超出显示省略号），标签不换行（溢出折叠处理）。

## 3. 功能模块详细说明

### 3.1 顶部操作区与高级筛选
1.  **视图模式**：默认且仅保留网格（卡片）视图模式。
2.  **扁平化高级筛选**：
    *   顶部筛选区采用 4 列网格化布局，默认直接展示两行核心筛选项，并在底部保留“展开 / 收起”、“清空”、“查询”操作按钮。
    *   默认展示的筛选项包括：素材名称、媒体、状态、素材类型、项目、设计师、类型标签、内容标签。
    *   点击“展开”后，继续显示扩展筛选项：优化师、创建时间、使用数据。
    *   **筛选条件包含**：
        *   媒体（下拉单选：头条、广点通、快手）
        *   状态（下拉单选：未使用、已投放、无消耗、有消耗、待审核、审核通过、审核失败）
        *   上传任务状态需在任务中心按状态机展示并支持筛选：`待执行`、`上传中`、`上传成功`、`同质化检测中`、`检测未通过`、`待审核`、`审核不通过`、`待入库`、`入库中`、`入库失败`、`已完成`、`已删除`。
        *   视频合成任务状态需在任务中心按状态机展示并支持筛选：`等待中`、`已暂停`、`同质化检测中`、`同质化合格`、`同质化过高`、`上传中`、`上传成功`、`上传失败`、`已删除`。
        *   素材类型（下拉单选：图片、视频）
        *   项目（通过项目选择弹窗进行多选）
        *   设计师（下拉单选：设计师A、设计师B）
        *   类型标签（点击唤出树状弹窗，支持单选，外部展示单个已选项并支持快速清空）
        *   内容标签（点击唤出树状弹窗，支持多选，外部展示已选项，超长显示 `+N` 并支持悬浮全览及快速清空）
        *   优化师（下拉单选：优化师A、优化师B；仅在展开状态展示）
        *   创建时间（自定义组件，点击呼出双面板日历弹窗及快捷范围如“近7天”；仅在展开状态展示）
        *   使用数据（消耗及 ROI 的数值范围筛选，支持选择 `>=`, `<=`, `=` 并输入具体数值；仅在展开状态展示）
    *   **占位符交互**：筛选项未选择时提示语显示浅灰色，一旦选中实际值，文本变为正常深色。

### 3.2 左侧分类树 (Sidebar)
1.  **文件夹搜索**：顶部提供一个附带放大镜 Icon 的输入框，用于快速搜索定位文件夹。
2.  **根目录分类**：分为三大核心顶层目录：**置顶**、**收藏**（内置“默认收藏夹”）、**全部**。
3.  **新建/管理操作交互**：
    *   **对于顶层目录**：鼠标悬停 (`hover`) 时，右侧显示加号 `+`，点击呼出“新建文件夹”模态框。弹窗内显示当前所在目录路径，并允许输入新名称。
    *   **对于子文件夹**：鼠标悬停 (`hover`) 时，右侧显示竖向省略号 `···`，点击展开下拉菜单。菜单内包含：新建文件夹、置顶、删除文件夹及其素材（标红警示）。

### 3.3 素材卡片 (Asset Card)
1.  **状态角标**：卡片右上角使用带有背景色的文字标签代替原本的圆点指示器（例如：灰底“未投放”、橙底“无消耗”、蓝底“有消耗”）。
2.  **标签溢出折叠**：底部属性标签（如“类型标签”、“内容标签”）超出卡片宽度时，自动隐藏多余标签并在末尾生成灰底的 `+X` 计数标签。鼠标悬浮在该区域时，通过自定义的黑色半透明上浮 Tooltip 完整显示所有标签。
3.  **卡片点击与详情查看**：
    *   点击素材卡片主体区域时，在系统框架中打开 `素材管理/素材详情.html` 新标签页。
    *   点击卡片底部“编辑”图标时，也需打开同一个“素材详情”页面，保持与卡片主体点击一致。
    *   素材详情页需采用独立分区排版，而不是简单的单列字段堆叠。
    *   页面顶部需显示“素材详情”标题，并保留关闭及编辑等操作入口。
4.  **媒体预览**：
    *   在详情抽屉首屏左侧提供素材预览区，用于承载图片或视频预览。
    *   点击图片素材：打开大图灯箱预览。
    *   点击视频素材：在弹窗中加载并播放视频。
5.  **悬浮操作栏 (Hover Actions)**：
    *   鼠标移至卡片上时显示底部操作图标，从左至右依次为：**下载**、**编辑**、**入库**（带下拉菜单：入库到待优化、入库到回收站）、**收藏**、**移动**。
    *   点击“下载”后应创建下载任务，并将处理进度统一沉淀到 `素材管理/我的任务.html` 的「下载」视图中查看。
    *   下载任务列表至少需展示：文件名称、文件类型、文件属性、文件大小、所属文件夹、所属系列/产品、下载进度、状态、提交时间、操作。
    *   点击“删除/回收站”等敏感选项时，均需触发“二次确认”弹窗。

### 3.4 素材详情页 (Asset Detail Page)
1.  **整体形态**：
    *   详情以独立页面形式存在，通过系统框架新标签页打开，不再以内嵌右侧抽屉承载。
    *   详情页内容允许页面级纵向滚动；标题区固定在顶部，避免滚动后丢失上下文。
2.  **首屏布局**：
    *   首屏采用左右两栏布局：左侧为“素材预览”，右侧为“素材信息（基础信息）”。
    *   左侧预览区宽度固定，右侧基础信息区自适应剩余宽度。
    *   右侧基础信息区优先使用两列属性卡 / 属性组排版，避免长表单式单列堆叠。
3.  **信息分区**：
    *   第二个主分区为“数据总览”，用于展示最关键的投放与表现指标。
    *   第三个主分区为“媒体信息”，通过平台 Tabs 切换查看不同媒体的数据与状态。
    *   媒体信息中每个平台至少覆盖审核信息、投放表现、评论 / 互动反馈或平台建议等内容块。
4.  **超长内容处理**：
    *   标签区默认限制展示高度，超出后通过“展开全部标签”查看完整内容。
    *   基础信息和数据总览在首屏只展示高优先级内容，其余信息通过“展开更多”逐步揭示。
    *   单个平台下内容过多时，优先在当前平台内容区内部滚动，不破坏整体抽屉节奏。
5.  **交互要求**：
    *   平台 Tabs 切换时，仅切换媒体信息区内容，不影响抽屉其余分区的滚动位置。
    *   抽屉关闭时返回素材库上下文，不改变当前筛选、排序与分页状态。

### 3.5 底部操作控制与多选模式 (Multi-Select)
1.  **默认状态**：右下角提供**选择**（进入多选模式按钮）、排序下拉框（按时间新旧、按名称A-Z）、**上传素材**（蓝色主按钮，点击跳转）。
2.  **进入多选模式**：
    *   点击“选择”后，所有素材卡片左上角出现复选框，但**默认不选中任何项**。
    *   底部操作栏变更为批量操作工具条。
    *   默认态下点击右上角“上传素材”按钮，需在系统框架内打开 `素材管理/上传素材.html` 新标签页。
3.  **多选工具条交互**：
    *   按钮排布顺序（从左至右）：`全选/取消全选`、`移动`、`收藏`、`转移到待优化库`、`删除`（红色）、`取消多选`（蓝色主按钮）。
    *   多选工具条中不再提供“入库”按钮，避免与卡片级入库动作重复。
    *   点击“移动”，弹出文件树结构模态框供用户选择目标路径。
    *   点击“删除”，弹出包含已选中数量的二次确认弹窗。
    *   点击“取消多选”，退出该模式，隐去复选框。
